# 01 安装与环境配置

## 前置要求

### 硬件要求
- **NVIDIA GPU**：需要支持 CUDA 的 NVIDIA 显卡（建议 Compute Capability 7.0+）
- **AMD GPU**：支持 HIP 后端（ROCm，如 MI300 系列）
- **Apple Silicon**：支持 Metal 后端（M1/M2/M3/M4 系列 Mac）

### 软件要求

| 要求 | 版本 |
|------|------|
| glibc | >= 2.28（Ubuntu 20.04+） |
| Python | >= 3.9 |
| CUDA | >= 10.0（或使用 pip 提供的 CUDA 工具链 >= 13.0） |

## 安装方式

### 方式一：通过 pip 安装（推荐）

```bash
pip install tilelang
```

或安装最新开发版：

```bash
pip install git+https://github.com/tile-ai/tilelang.git
```

或安装 nightly 版本：

```bash
pip install tilelang -f https://tile-ai.github.io/whl/nightly
```

### 方式二：从源码编译安装

```bash
# 克隆仓库（注意 --recursive 包含 TVM 子模块）
git clone --recursive https://github.com/tile-ai/tilelang.git
cd tilelang

# 编译安装
pip install . -v
```

**开发模式安装**（推荐开发者使用）：

```bash
pip install -e . -v
```

### 方式三：使用 pip 提供的 CUDA 工具链（无需本地安装 CUDA）

```bash
git clone --recursive https://github.com/tile-ai/tilelang.git
cd tilelang
pip install -r requirements-dev.txt
pip install "nvidia-cuda-nvcc>=13" "nvidia-cuda-cccl>=13" "nvidia-cuda-nvrtc>=13"
pip install . -v --no-build-isolation
```

### 方式四：通过 PYTHONPATH 开发（推荐核心开发者）

```bash
git clone --recursive https://github.com/tile-ai/tilelang.git
cd tilelang
mkdir build && cd build
cmake .. -DUSE_CUDA=ON -G Ninja
ninja

# 设置环境变量
export PYTHONPATH=/path/to/tilelang:$PYTHONPATH
python -c "import tilelang; print(tilelang.__version__)"
```

### 方式五：Docker 安装

```bash
git clone --recursive https://github.com/tile-ai/tilelang.git
cd tilelang/docker
docker build -f Dockerfile.cu120 -t tilelang-cu120 .

# 运行容器
docker run -itd \
  --shm-size 32g \
  --gpus all \
  -v /home/tilelang:/home/tilelang \
  --name tilelang_dev \
  tilelang-cu120 \
  /bin/zsh
```

### AMD ROCm 安装（MI300 系列）

```bash
git clone --recursive https://github.com/tile-ai/tilelang.git /opt/tilelang
cd /opt/tilelang
export CMAKE_ARGS="-DUSE_CUDA=OFF -DUSE_ROCM=ON -DROCM_PATH=/opt/rocm"
pip install -e . -v --no-build-isolation --no-deps
pip install "apache-tvm-ffi>=0.1.6" "z3-solver>=4.13.0"
```

### CMake 编译选项

| 选项 | 默认值 | 说明 |
|------|--------|------|
| `-DUSE_CUDA` | ON (Linux) | 启用 CUDA 支持 |
| `-DUSE_ROCM` | OFF | 启用 ROCm 支持 |
| `-DUSE_METAL` | ON (macOS) | 启用 Metal 支持 |
| `TVM_ROOT` | - | 自定义 TVM 源码路径 |

## 编译目标（Target）

TileLang 构建在 TVM 之上，通过 **Target** 指定编译目标设备：

### 常用 Target 字符串

| Target | 说明 |
|--------|------|
| `"auto"` | 自动检测 CUDA → HIP → Metal |
| `"cuda"` | NVIDIA GPU |
| `"cuda -arch=sm_90"` | 指定特定架构 |
| `"hip"` | AMD GPU（ROCm） |
| `"metal"` | Apple Silicon |
| `"llvm"` | CPU 执行 |
| `"c"` | 生成 C 源码（用于检查） |

### 常见 GPU 架构对照

| 架构 | GPU 示例 |
|------|----------|
| `sm_75` | T4, RTX 2080 |
| `sm_80` | A100, A30 |
| `sm_86` | A40, A10, RTX 3090 |
| `sm_89` | RTX 4090 |
| `sm_90a` | H100 |
| `sm_100a` | B100 |

### 使用方式

```python
# 方式 1：在 jit 中指定
@tilelang.jit(out_idx=[2], target="cuda -arch=sm_80")
def my_kernel(...):
    ...

# 方式 2：在 compile 中指定
program = tilelang.compile(func, target="cuda -arch=sm_90", execution_backend="cython")

# 方式 3：编程式创建
from tilelang.utils.target import determine_target
target = determine_target("cuda -arch=sm_80", return_object=True)
```

## 编译缓存

TileLang 默认将编译缓存存储在 `~/.tilelang/cache`，重复调用相同参数的内核会直接使用缓存。如需清理：

```python
import tilelang
tilelang.clear_cache()
```

## 第一个 TileLang 程序

```python
import tilelang
import tilelang.language as T
import torch

@tilelang.jit(out_idx=[2])
def vector_add(N: int, block: int = 256, dtype: str = "float32"):
    @T.prim_func
    def main(
        A: T.Tensor((N,), dtype),
        B: T.Tensor((N,), dtype),
        C: T.Tensor((N,), dtype),
    ):
        with T.Kernel(T.ceildiv(N, block), threads=block) as bx:
            for i in T.Parallel(block):
                gi = bx * block + i
                C[gi] = A[gi] + B[gi]
    return main

# 测试
N = 1024
A = torch.randn(N, device="cuda", dtype=torch.float32)
B = torch.randn(N, device="cuda", dtype=torch.float32)
C = torch.empty(N, device="cuda", dtype=torch.float32)

kernel = vector_add(N)
kernel(A, B, C)

torch.testing.assert_close(C, A + B)
print("向量加法验证通过！")
```

## 常见问题

### Q: 编译时间较长？
首次编译需要生成和优化 CUDA/HIP 代码，后续调用使用缓存会快很多。

### Q: 如何查看生成的内核代码？
```python
program = tilelang.compile(func, target="cuda")
print(program.get_kernel_source())
```

### Q: 运行时报 "no kernel image is available"？
`-arch` 参数与实际 GPU 不匹配，请确认 GPU 架构并指定正确的 target。
