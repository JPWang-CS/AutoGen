# 02 语言基础

## 编程模型概述

TileLang 采用 **基于 Tile 的编程模型**，开发者通过定义 **Tile（数据块）** 级别的操作来编写高性能内核。核心思路是：

1. 将全局数据划分为多个 Tile
2. 每个 Thread Block 负责处理一个或多个 Tile
3. 利用 GPU 的三级内存层次进行高效计算

### 三级内存层次

```
┌─────────────────────────────────────────┐
│           Global Memory (全局内存)        │  大容量、高延迟
│   所有 Thread Block 共享，通过 T.copy 访问  │
├─────────────────────────────────────────┤
│           Shared Memory (共享内存)        │  中等容量、低延迟
│   同一 Thread Block 内所有线程共享         │  通过 T.alloc_shared 分配
├─────────────────────────────────────────┤
│        Fragment / Register (寄存器)      │  最小容量、最低延迟
│   每个线程私有（或 Warp 级别共享）          │  通过 T.alloc_fragment 分配
└─────────────────────────────────────────┘
```

## 内核定义：`@T.prim_func`

所有 TileLang 内核都通过 `@T.prim_func` 装饰器定义：

```python
import tilelang.language as T

@T.prim_func
def my_kernel(
    A: T.Tensor((M, K), "float16"),  # 输入张量 A
    B: T.Tensor((K, N), "float16"),  # 输入张量 B
    C: T.Tensor((M, N), "float16"),  # 输出张量 C
):
    # 内核逻辑
    ...
```

**接口说明**：
- `T.Tensor(shape, dtype)`：声明张量参数，指定形状和数据类型
- 函数体内的代码会被 TileLang 编译器转换为 GPU 内核代码
- 张量形状可以使用外部变量（如 `M`、`N`、`K`），但需要通过闭包或 JIT 函数参数传入

## 内核入口：`T.Kernel`

`T.Kernel` 定义了内核的并行维度和线程配置：

```python
with T.Kernel(grid_x, grid_y, grid_z, threads=num_threads) as (bx, by, bz):
    # bx, by, bz 分别是 blockIdx.x, blockIdx.y, blockIdx.z
    ...
```

### 接口参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `*grid_size` | int | Grid 的各维度大小（可传 1~3 个整数） |
| `threads` | int | 每个 Thread Block 的线程数（必须为 32 的倍数） |

### 返回值

返回一个元组 `(bx, by, bz)`，每个元素代表对应维度的 Block 索引。

### 示例

```python
# 1D Grid
with T.Kernel(T.ceildiv(N, BN), threads=128) as bx:
    ...

# 2D Grid（最常用）
with T.Kernel(T.ceildiv(N, BN), T.ceildiv(M, BM), threads=128) as (bx, by):
    ...

# 3D Grid
with T.Kernel(T.ceildiv(N, BN), T.ceildiv(M, BM), T.ceildiv(K, BK), threads=128) as (bx, by, bz):
    ...
```

> **注意**：`T.ceildiv(a, b)` 是向上取整除法，等价于 `(a + b - 1) // b`，常用于计算需要的 Block 数量。

## 内存分配

### T.alloc_shared — 分配共享内存

```python
A_shared = T.alloc_shared((BM, BK), "float16")
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `shape` | tuple | 共享内存缓冲区的形状 |
| `dtype` | str | 数据类型 |

- **作用域**：同一 Thread Block 内所有线程共享
- **生命周期**：Kernel 执行期间有效
- **容量限制**：通常 48KB~96KB（取决于 GPU 架构）

### T.alloc_fragment — 分配 Fragment/寄存器内存

```python
C_frag = T.alloc_fragment((BM, BN), "float32")
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `shape` | tuple | Fragment 缓冲区的形状 |
| `dtype` | str | 数据类型 |

- **作用域**：线程级私有（通常用于矩阵乘累加器）
- **生命周期**：Kernel 执行期间有效
- **特点**：常用于存放 `T.gemm` 的累加结果，支持高效寄存器级运算

### T.alloc_var — 分配局部变量

```python
tid = T.alloc_var("int32")
tid = threadIdx.x
```

用于分配单个标量变量，常用于存储线程索引等中间值。

## 数据搬运：T.copy

`T.copy` 是最基本的数据搬运指令，用于在 Global、Shared、Fragment 内存之间传输数据：

```python
# Global → Shared
T.copy(A[by * BM, ko * BK], A_shared)

# Shared → Fragment（通过 T.gemm 间接使用）

# Fragment → Global
T.copy(C_frag, C[by * BM, bx * BN])
```

> 详细用法请参考 [03 核心指令详解](03_核心指令详解.md)

## 完整示例：向量加法内核

```python
import tilelang
import tilelang.language as T

@tilelang.jit(out_idx=[2])
def vec_add(N: int, BN: int = 128, dtype: str = "float16"):
    @T.prim_func
    def main(
        A: T.Tensor((N,), dtype),
        B: T.Tensor((N,), dtype),
        C: T.Tensor((N,), dtype),
    ):
        # 计算需要的 Block 数量
        with T.Kernel(T.ceildiv(N, BN), threads=BN) as bx:
            # 每个 Block 处理 BN 个元素
            # T.copy 会自动将数据从 Global 搬运到寄存器
            A_local = T.alloc_fragment((BN,), dtype)
            B_local = T.alloc_fragment((BN,), dtype)
            C_local = T.alloc_fragment((BN,), dtype)

            T.copy(A[bx * BN], A_local)
            T.copy(B[bx * BN], B_local)

            # 逐元素相加
            for i in T.serial(BN):
                C_local[i] = A_local[i] + B_local[i]

            T.copy(C_local, C[bx * BN])

    return main
```

## JIT 编译与执行：@tilelang.jit

`@tilelang.jit` 是最常用的编译执行方式：

```python
@tilelang.jit(out_idx=[2])
def my_kernel_func(M, N, block_size=128, dtype="float16"):
    @T.prim_func
    def kernel(A: T.Tensor((M, N), dtype), B: T.Tensor((M, N), dtype), C: T.Tensor((M, N), dtype)):
        ...
    return kernel
```

### 接口参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `out_idx` | list[int] | 输出张量在参数列表中的索引（支持 `-1` 表示最后一个） |
| `target` | str | 编译目标：`"cuda"`、`"hip"`、`"metal"` |
| `executable` | bool | 是否返回可执行对象 |

### 调用方式

```python
# 编译时只传配置参数
kernel = my_kernel_func(M=1024, N=1024)

# 运行时传入张量
import torch
A = torch.randn(1024, 1024, device="cuda", dtype=torch.float16)
B = torch.randn(1024, 1024, device="cuda", dtype=torch.float16)
C = kernel(A, B)  # 自动执行并返回 C
```

## Python 兼容性限制

TileLang DSL **不支持**所有 Python 特性。以下是支持和不支持的特性对比：

### 支持的特性

| 特性 | 示例 |
|------|------|
| 基本算术运算 | `a + b`, `a * b`, `a / b` |
| 比较运算 | `a > b`, `a <= b`, `a == b` |
| 变量赋值 | `x = 5` |
| for 循环（通过 T.serial 等） | `for i in T.serial(N):` |
| if/else 条件 | `if condition: ... else: ...` |
| Tensor 索引 | `A[i, j]` |

### 不支持的特性

| 特性 | 替代方案 |
|------|----------|
| `def` 函数定义 | 使用 `@T.macro` |
| `class` 类定义 | 无，使用结构化数据 |
| `enumerate` / `zip` | 手动管理索引 |
| 列表推导式 | 使用 `T.serial` 循环 |
| `print` | 使用调试工具 |
| `import` | 在 Kernel 外导入 |
| `try/except` | 不支持异常处理 |

### @T.macro — 可复用代码块

由于不支持函数定义，TileLang 提供 `@T.macro` 用于定义可复用的代码片段：

```python
@T.macro
def copy_and_compute(A, B, C, offset):
    T.copy(A[offset], B)
    for i in T.serial(128):
        C[i] = B[i] * 2.0

# 在 Kernel 中使用
with T.Kernel(...) as (bx,):
    A_s = T.alloc_shared((128,), "float16")
    C_f = T.alloc_fragment((128,), "float16")
    copy_and_compute(A, A_s, C_f, bx * 128)
```
