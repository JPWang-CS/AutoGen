# 03 核心指令详解

本章详细介绍 TileLang 提供的所有核心指令（Instructions），包括数据搬运、计算、内存管理、同步、原子操作等。

---

## 一、数据搬运指令

### 1.1 T.copy — 同步数据拷贝

`T.copy` 是最基础的数据搬运指令，用于在不同内存层级之间高效传输数据。

```python
T.copy(src, dst)
```

**功能**：将数据从 `src` 拷贝到 `dst`，支持 Global ↔ Shared ↔ Fragment 之间的任意传输。

**支持的拷贝路径**：

| 源 → 目标 | 说明 |
|-----------|------|
| Global → Shared | 从全局内存加载到共享内存 |
| Shared → Global | 从共享内存写回全局内存 |
| Global → Fragment | 从全局内存直接加载到寄存器 |
| Fragment → Global | 从寄存器写回全局内存 |
| Shared ↔ Fragment | 共享内存和寄存器之间的传输 |

**示例**：

```python
# Global → Shared：从全局内存加载矩阵块到共享内存
T.copy(A[by * BM, ko * BK], A_shared)

# Fragment → Global：将累加结果写回全局内存
T.copy(C_frag, C[by * BM, bx * BN])

# 带偏移的拷贝
T.copy(A[block_row * BM + ko * BK, 0], A_shared)
```

**特点**：
- 自动利用最优的内存访问模式（如向量化加载、合并访问）
- 对于 Global → Shared 的拷贝，会自动使用最优的加载策略
- 所有线程协同完成拷贝，无需手动管理线程索引

**可选参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `coalesced_width` | int | 控制内存合并访问宽度 |
| `disable_tma` | bool | 禁用 TMA 加速 |
| `eviction_policy` | str | 缓存逐出策略 |
| `loop_layout` | Fragment | 循环布局注解 |

---

### 1.2 T.async_copy — 异步数据拷贝

```python
T.async_copy(src, dst)
```

**功能**：使用 GPU 的异步拷贝引擎（如 `cp.async`）进行数据传输，允许数据拷贝与计算重叠执行。

**与 T.copy 的关键区别**：
- `T.copy`：同步语义，拷贝完成后才继续执行（编译器可能使用异步机制但保证同步语义）
- `T.async_copy`：显式异步，发起拷贝后立即返回，**必须**手动等待完成

**使用模式**：

```python
# 异步预取
T.async_copy(A[by * BM, ko * BK], A_s)

# ... 可以执行不依赖 A_s 的其他工作 ...

# 使用前必须等待异步拷贝完成
T.ptx_wait_group(0)

# 然后可以安全使用 A_s
T.gemm(A_s, B_s, C_f)
```

**重要约束**：
- 如果拷贝不能降级为 `cp.async`（如错误的作用域、不支持的向量宽度），编译会**直接失败**而非静默回退
- 跨线程消费 `dst` 时仍需要 shared memory barrier
- 在大多数 TileLang 程序中，`ThreadSync("shared")` 会在首次读取 `dst` 前自动插入所需的 `T.tvm_storage_sync("shared")`

---

### 1.3 T.c2d_im2col — 卷积 im2col 转换

```python
T.c2d_im2col(src, dst, ...)
```

**功能**：将卷积输入数据从 NCHW 格式转换为 im2col 格式，用于将卷积运算转换为矩阵乘法。

**参数说明**：
- `src`：输入特征图（Global Memory）
- `dst`：输出 im2col 矩阵（通常为 Shared Memory）
- 额外参数包括卷积核大小、步长、填充等

## 二、计算指令

### 2.1 T.gemm — 矩阵乘法

```python
T.gemm(A, B, C)
```

**功能**：执行矩阵乘法并累加 `C += A @ B`。

**参数**：

| 参数 | 说明 | 内存位置 |
|------|------|----------|
| `A` | 左操作数矩阵 | Shared Memory 或 Fragment |
| `B` | 右操作数矩阵 | Shared Memory 或 Fragment |
| `C` | 累加结果矩阵 | Fragment（必须） |

**约束**：
- `C` 必须是 Fragment 类型（寄存器内存）
- `A` 和 `B` 通常为 Shared Memory
- 内部使用 Tensor Core（如果可用）加速计算
- 支持混合精度：`A`、`B` 可以是 `float16`/`bfloat16`，`C` 可以是 `float32`

**示例**：

```python
A_s = T.alloc_shared((BM, BK), "float16")
B_s = T.alloc_shared((BK, BN), "float16")
C_f = T.alloc_fragment((BM, BN), "float32")

# C_f += A_s @ B_s
T.gemm(A_s, B_s, C_f)
```

**注意事项**：
- `T.gemm` 执行的是 **累加** 操作，不会清零 `C`。如果需要从零开始累加，请先使用 `T.clear(C_f)` 清零
- 输入矩阵的形状必须满足矩阵乘法的维度匹配规则

---

### 2.2 T.gemm_sp — 稀疏矩阵乘法

```python
T.gemm_sp(A, B, C, ...)
```

**功能**：执行稀疏矩阵乘法，用于结构化稀疏场景（如 2:4 稀疏模式）。

**适用场景**：
- 模型剪枝后的稀疏推理
- 支持稀疏训练场景
- 利用 NVIDIA Ampere+ 架构的稀疏 Tensor Core

---

### 2.3 逐元素数学运算

TileLang 支持丰富的逐元素数学函数，可以直接在 Fragment 上操作：

```python
# 指数和对数
T.exp(x)        # e^x
T.log(x)        # ln(x)
T.log2(x)       # log2(x)

# 幂运算
T.sqrt(x)       # √x
T.rsqrt(x)      # 1/√x（比 1/sqrt(x) 更高效）

# 取极值
T.max(a, b)     # max(a, b)
T.min(a, b)     # min(a, b)

# 取反与绝对值
T.abs(x)        # |x|
T.neg(x)        # -x

# 其他
T.sigmoid(x)    # 1 / (1 + exp(-x))
T.tanh(x)       # tanh(x)
T.clamp(x, lo, hi)  # 将 x 限制在 [lo, hi] 范围内
T.cast(x, dtype)    # 类型转换
```

**使用示例**：

```python
C_frag = T.alloc_fragment((BM, BN), "float32")

# 对 Fragment 中的每个元素应用 exp
for i in T.serial(BM):
    for j in T.serial(BN):
        C_frag[i, j] = T.exp(C_frag[i, j])
```

---

### 2.4 归约运算

```python
# 求和归约
result = T.reduce_sum(buffer, axis=0)

# 最大值归约
result = T.reduce_max(buffer, axis=0)

# 最小值归约
result = T.reduce_min(buffer, axis=0)
```

**参数**：

| 参数 | 说明 |
|------|------|
| `buffer` | 输入缓冲区（Fragment 或 Shared） |
| `axis` | 归约的轴 |

**示例**：在线 Softmax 中的归约

```python
# 计算 softmax 的分母（每行求和）
row_sum = T.alloc_fragment((1,), "float32")
row_sum[0] = T.reduce_sum(score_frag, axis=1)
```

## 三、内存管理指令

### 3.1 T.clear — 清零缓冲区

```python
T.clear(buffer)
```

**功能**：将缓冲区的所有元素设置为 0。

**示例**：

```python
C_frag = T.alloc_fragment((BM, BN), "float32")
T.clear(C_frag)  # 将所有元素清零
```

**使用场景**：在矩阵乘法累加之前，需要先清零累加器。

---

### 3.2 T.fill — 填充缓冲区

```python
T.fill(buffer, value)
```

**功能**：将缓冲区的所有元素填充为指定值。

**示例**：

```python
bias_frag = T.alloc_fragment((BM, BN), "float32")
T.fill(bias_frag, 1.0)  # 填充为 1.0
```

---

### 3.3 T.alloc_barrier — 分配同步屏障

```python
barrier = T.alloc_barrier()
```

**功能**：分配一个线程同步屏障，用于 Thread Block 内的线程同步。

---

### 3.4 T.alloc_tmem — 分配 Tensor Memory（Blackwell 架构）

```python
tmem = T.alloc_tmem(shape, dtype)
```

**功能**：分配 Tensor Memory，这是 NVIDIA Blackwell 架构（SM100）引入的新内存层级。

**适用场景**：仅适用于 Blackwell 及以上架构的 GPU。

---

## 四、Tensor 操作指令

### 4.1 T.reshape — 改变形状

```python
reshaped = T.reshape(buffer, new_shape)
```

**功能**：改变缓冲区的视图形状，不复制数据。

**示例**：

```python
A_flat = T.alloc_fragment((BM * BK), "float16")
A_2d = T.reshape(A_flat, (BM, BK))
```

---

### 4.2 T.view — 创建视图

```python
viewed = T.view(buffer, dtype)
```

**功能**：以不同的数据类型查看同一块内存，不复制数据（类似 C 语言的 reinterpret_cast）。

**示例**：

```python
# 将 float16 视图转换为 uint16（用于位操作）
data_f16 = T.alloc_shared((128,), "float16")
data_u16 = T.view(data_f16, "uint16")
```

---

## 五、同步指令

### 5.1 T.barrier_sync — 屏障同步

```python
# 使用分配的屏障进行同步
barrier = T.alloc_barrier()
# ... 线程执行一些操作 ...
T.barrier_sync(barrier)  # 等待所有线程到达此点
```

**功能**：Thread Block 内所有线程在此处等待，直到所有线程都到达同步点。

### 5.2 T.wait_async — 等待异步操作完成

```python
T.wait_async()
```

**功能**：等待之前发起的所有 `T.async_copy` 操作完成。

## 六、原子操作

TileLang 提供了 GPU 原子操作，用于多线程间的安全数据更新：

```python
# 原子加
T.atomic_add(dst, value)

# 原子最大值
T.atomic_max(dst, value)

# 原子最小值
T.atomic_min(dst, value)

# 原子交换
T.atomic_exchange(dst, value)

# 原子比较并交换
T.atomic_cas(dst, expected, desired)
```

**使用示例**：

```python
# 多个 Block 向全局计数器累加
counter = T.alloc_var("int32")
T.atomic_add(counter, 1)
```

---

## 七、TMA（Tensor Memory Accelerator）操作

TMA 是 NVIDIA Hopper 架构引入的硬件加速数据搬运功能：

```python
# TMA 描述符声明
T.tma_decl(tensor, shape, dtype, ...)

# TMA 异步加载
T.tma_load(desc, dst, coordinates...)

# TMA 存储
T.tma_store(desc, src, coordinates...)
```

**适用场景**：
- 大块数据的高效搬运
- 多维 Tensor 的分块加载
- Hopper（SM90）及以上架构

---

## 八、Warp Group 操作

Warp Group 是协作执行的 4 个 Warp（128 个线程）：

```python
# Warp Group 级别的矩阵乘法
T.wg_gemm(A, B, C)

# Warp Group 的 MMA 操作
T.wg_mma(A, B, C)
```

**适用场景**：Hopper 及以上架构的大规模矩阵运算。

## 九、诊断与调试

### 9.1 T.print — 调试打印

```python
T.print(value)  # 在内核执行时打印值
```

### 9.2 T.device_assert — 设备端断言

```python
T.device_assert(condition, msg='')
```

**功能**：在 CUDA 目标上发射设备端断言。用于调试和验证内核逻辑。

> **注意**：`T.print` 和 `T.device_assert` 仅用于调试，可能影响内核性能。

## 十、注解与辅助指令

### 10.1 T.use_swizzle — 光栅化提示

```python
T.use_swizzle(panel_size=10, enable=True)
```

**功能**：提供光栅化（rasterization）提示，优化 Thread Block 的调度顺序以改善缓存命中率。

### 10.2 T.annotate_layout — 布局注解

```python
T.annotate_layout({buffer: layout_fragment})
```

**功能**：为 buffer 附加显式布局信息。

### 10.3 T.annotate_safe_value — 安全值提示

```python
T.annotate_safe_value(var, value)
```

**功能**：提供安全性或常量提示，帮助编译器优化。

### 10.4 T.annotate_l2_hit_ratio — L2 缓存提示

```python
T.annotate_l2_hit_ratio(buffer, ratio)
```

**功能**：提示 L2 缓存的预期命中率，影响数据预取策略。

### 10.5 逻辑辅助

```python
# 多条件 OR
if T.any_of(cond1, cond2, cond3):
    ...

# 多条件 AND
if T.all_of(i < M, j < N):
    ...
```
