# 03 核心指令详解

本章详细介绍 TileLang-Ascend 提供的所有核心指令，包括内存分配、数据搬运、矩阵计算、向量计算、同步等。

---

## 一、内存分配指令

### 1.1 Developer模式分配

#### T.alloc_shared — 分配片上共享存储

```python
buffer = T.alloc_shared(shape, dtype)
```

编译器根据上下文自动映射到L1 Buffer（Cube操作场景）或Unified Buffer（Vector操作场景）。

**示例**：
```python
A_L1 = T.alloc_shared((block_M, block_K), "float16")
C_L0 = T.alloc_fragment((block_M, block_N), "float")
```

#### T.alloc_fragment — 分配Fragment存储

```python
buffer = T.alloc_fragment(shape, dtype)
```

编译器自动映射到L0C或其他寄存器级存储。在GEMM场景中，用于存放矩阵乘累加结果。

**示例**：
```python
C_L0 = T.alloc_fragment((block_M, block_N), "float")
```

### 1.2 Expert模式分配

#### T.alloc_L1 — 分配L1 Buffer

```python
A_L1 = T.alloc_L1((block_M, K_L1), "float16")
```

L1 Buffer是Cube Core的高速缓存，容量通常为1MB（910B），用于矩阵乘法的输入暂存。

#### T.alloc_ub — 分配Unified Buffer

```python
a_ub = T.alloc_ub((block_M // VEC_NUM, block_N), "float32")
```

Unified Buffer是Vector Core的计算缓冲区，容量通常为2MB（910B）。所有 `T.tile.*` 向量操作都在UB上进行。

**注意**：分配形状需要按 `VEC_NUM=2` 均分，即第一维度为 `block_M // VEC_NUM`。

#### T.alloc_L0A / T.alloc_L0B — 分配矩阵输入寄存器

```python
A_L0 = T.alloc_L0A((block_M, block_K), "float16")  # 左矩阵
B_L0 = T.alloc_L0B((block_K, block_N), "float16")  # 右矩阵
```

L0A/L0B是Cube Core的矩阵输入寄存器，分别存放矩阵乘法的左矩阵和右矩阵。

#### T.alloc_L0C — 分配矩阵输出寄存器

```python
C_L0 = T.alloc_L0C((block_M, block_N), "float32")  # 累加结果
```

L0C是Cube Core的矩阵输出寄存器，存放矩阵乘法的累加结果。数据类型通常为 `float32`（`"float"`）。

#### T.alloc_var — 分配标量变量

```python
# 基本用法
flag = T.alloc_var("bool", init=False)
counter = T.alloc_var("int32", init=1)
value = T.alloc_var("float32", init=0.0)

# 变量间初始化
a = T.alloc_var("int32", init=1)
b = T.alloc_var("int32", init=a)
```

## 二、数据搬运指令

### 2.1 T.copy — 数据拷贝

```python
T.copy(src, dst)
```

**功能**：将数据从 `src` 搬运到 `dst`，支持不同内存层级之间的数据传输。

**支持的数据搬运路径**：

| 源 → 目标 | 说明 | 典型作用域 |
|-----------|------|-----------|
| GM → L1 | 从全局内存加载到L1 Buffer | `T.Scope("C")` |
| L1 → L0A | 从L1搬运到Cube左矩阵输入 | `T.Scope("C")` |
| L1 → L0B | 从L1搬运到Cube右矩阵输入 | `T.Scope("C")` |
| L0C → GM | 从Cube输出搬运到全局内存 | `T.Scope("C")` |
| GM → UB | 从全局内存加载到Unified Buffer | `T.Scope("V")` |
| UB → GM | 从Unified Buffer写回全局内存 | `T.Scope("V")` |
| UB → UB | UB之间的数据拷贝 | `T.Scope("V")` |
| UB → L1 | 从UB搬运到L1 | `T.Scope("V")` |

**示例**：

```python
# GM -> L1：加载矩阵块到Cube缓存
T.copy(A[bx * block_M, k * K_L1], A_L1)

# GM -> UB：加载数据到Vector缓冲区
T.copy(A[bx * block_M + vid * block_M // VEC_NUM, by * block_N], a_ub)

# L0C -> GM：将矩阵乘结果写回
T.copy(C_L0, C[bx * block_M, by * block_N])

# UB -> GM：将向量计算结果写回
T.copy(c_ub, C[bx * block_M + vid * block_M // VEC_NUM, by * block_N])
```

## 三、矩阵计算指令

### 3.1 T.gemm_v0 — 矩阵乘法（高级接口）

```python
T.gemm_v0(A, B, C, transpose_A=False, transpose_B=False, init=False)
```

**功能**：执行矩阵乘法操作 `C += A @ B`（当 `init=True` 时先清零再计算，即 `C = A @ B`）。

**参数**：

| 参数 | 说明 |
|------|------|
| `A` | 左输入矩阵（L1 Buffer） |
| `B` | 右输入矩阵（L1 Buffer） |
| `C` | 累加结果矩阵（L0C Buffer） |
| `transpose_A` | 是否对A矩阵转置，默认 False |
| `transpose_B` | 是否对B矩阵转置，默认 False |
| `init` | 是否在计算前清零累加矩阵。首次计算时设为True |

**示例**：

```python
A_L1 = T.alloc_L1((block_M, K_L1), "float16")
B_L1 = T.alloc_L1((K_L1, block_N), "float16")
C_L0 = T.alloc_L0C((block_M, block_N), "float")

loop_k = T.ceildiv(K, K_L1)
for k in T.serial(loop_k):
    T.copy(A[bx * block_M, k * K_L1], A_L1)
    T.copy(B[k * K_L1, by * block_N], B_L1)

    T.barrier_all()
    T.gemm_v0(A_L1, B_L1, C_L0, init=(k == 0))  # 首次清零，后续累加
    T.barrier_all()
```

**累加器清零方式**：通过 `init` 参数控制。`init=True` 时清零后计算，`init=False` 时在已有值上累加。通常第一次迭代用 `init=True`，后续用默认值 `False`。

### 3.2 T.mma — 矩阵乘法（低级接口）

```python
T.mma(A_L0A, B_L0B, C_L0C, init=False)
```

**功能**：执行矩阵乘法操作，输入直接来自L0A/L0B寄存器，输出到L0C寄存器。

**参数**：

| 参数 | 说明 |
|------|------|
| `A_L0A` | 左矩阵（L0A Buffer） |
| `B_L0B` | 右矩阵（L0B Buffer） |
| `C_L0C` | 累加结果（L0C Buffer） |
| `init` | 是否清零累加矩阵 |

**与 T.gemm_v0 的区别**：`T.gemm_v0` 的输入在L1中，会自动完成L1→L0A/L0B的搬运和计算；`T.mma` 需要手动将数据从L1搬运到L0A/L0B，提供更精细的控制。

**示例**（Expert模式流水线）：

```python
A_L0 = T.alloc_L0A((S2, block_M, block_K), "float16")
B_L0 = T.alloc_L0B((S2, block_K, block_N), "float16")
C_L0 = T.alloc_L0C((block_M, block_N), "float")

# 手动搬运 L1 -> L0A/L0B
T.copy(A_L1[k % S1, 0, kk * block_K], A_L0[kk % S2, :, :])
T.copy(B_L1[k % S1, kk * block_K, 0], B_L0[kk % S2, :, :])

# 执行矩阵乘法
T.mma(A_L0[kk % S2, :, :], B_L0[kk % S2, :, :], C_L0, init=T.And(k == 0, kk == 0))
```

## 四、向量计算指令（T.tile.*）

所有向量计算指令操作Unified Buffer上的数据，必须在 `T.Scope("V")` 内使用。

### 4.1 算术运算

| 指令 | 功能 | 示例 |
|------|------|------|
| `T.tile.add(dst, src0, src1)` | 逐元素加法 dst = src0 + src1 | `T.tile.add(c_ub, a_ub, b_ub)` |
| `T.tile.sub(dst, src0, src1)` | 逐元素减法 dst = src0 - src1 | `T.tile.sub(c_ub, a_ub, b_ub)` |
| `T.tile.mul(dst, src0, src1)` | 逐元素乘法 dst = src0 * src1 | `T.tile.mul(c_ub, a_ub, b_ub)` |
| `T.tile.div(dst, src0, src1)` | 逐元素除法 dst = src0 / src1 | `T.tile.div(c_ub, a_ub, b_ub)` |
| `T.tile.max(dst, src0, src1)` | 逐元素取最大值 | `T.tile.max(c_ub, a_ub, b_ub)` |
| `T.tile.min(dst, src0, src1)` | 逐元素取最小值 | `T.tile.min(c_ub, a_ub, b_ub)` |

> `src1` 可以是buffer类型，也可以是标量（scalar）类型，例如 `T.tile.add(c_ub, a_ub, 2)`。

### 4.2 数学函数

| 指令 | 功能 | 示例 |
|------|------|------|
| `T.tile.exp(dst, src)` | 自然指数 dst = exp(src) | `T.tile.exp(c_ub, a_ub)` |
| `T.tile.ln(dst, src)` | 自然对数 dst = ln(src) | `T.tile.ln(c_ub, a_ub)` |
| `T.tile.abs(dst, src)` | 绝对值 dst = abs(src) | `T.tile.abs(c_ub, a_ub)` |
| `T.tile.sqrt(dst, src)` | 平方根 dst = sqrt(src) | `T.tile.sqrt(c_ub, a_ub)` |
| `T.tile.rsqrt(dst, src)` | 平方根倒数 dst = 1/sqrt(src) | `T.tile.rsqrt(c_ub, a_ub)` |
| `T.tile.reciprocal(dst, src)` | 倒数 dst = 1/src | `T.tile.reciprocal(c_ub, a_ub)` |
| `T.tile.sin(dst, src)` | 正弦 dst = sin(src) | `T.tile.sin(c_ub, a_ub)` |
| `T.tile.cos(dst, src)` | 余弦 dst = cos(src) | `T.tile.cos(c_ub, a_ub)` |

### 4.3 激活函数

| 指令 | 功能 | 示例 |
|------|------|------|
| `T.tile.relu(dst, src)` | ReLU: dst = max(0, src) | `T.tile.relu(c_ub, a_ub)` |
| `T.tile.sigmoid(dst, src)` | Sigmoid: dst = 1/(1+exp(-src)) | `T.tile.sigmoid(c_ub, a_ub)` |
| `T.tile.leaky_relu(dst, src, scalar)` | Leaky ReLU | `T.tile.leaky_relu(c_ub, a_ub, 0.01)` |
| `T.tile.axpy(dst, src, scalar)` | AXPY: dst = scalar*src + dst | `T.tile.axpy(c_ub, a_ub, 2.0)` |

> **注意**: `sigmoid`、`silu`、`tanh`、`gelu` 等激活函数可以：1) 使用内置 `T.tile.sigmoid()` 指令；2) 通过 `T.tile.exp`/`T.tile.div` 等基本指令手动组合实现。手动组合方式可参考 `tilelang-ascend/examples/activation/` 目录下的示例。

### 4.4 数据操作

| 指令 | 功能 | 示例 |
|------|------|------|
| `T.tile.fill(buffer, value)` | 用value填充buffer | `T.tile.fill(acc_o, 0.0)` |
| `T.tile.clear(buffer)` | 将buffer清零 | `T.tile.clear(ub)` |
| `T.tile.cast(dst, src, mode, count)` | 精度转换 | `T.tile.cast(b_ub, a_ub, "CAST_RINT", 4096)` |
| `T.tile.transpose(dst, src)` | 16x16矩阵转置 | `T.tile.transpose(b_ub, a_ub)` |

**cast模式**：
- `"CAST_NONE"`：无特殊舍入
- `"CAST_RINT"`：四舍五入到最近偶数
- `"CAST_FLOOR"`：向下取整
- `"CAST_CEIL"`：向上取整
- `"CAST_ROUND"`：四舍五入
- `"CAST_TRUNC"`：截断
- `"CAST_ODD"`：向奇数舍入

### 4.5 比较与选择

| 指令 | 功能 | 示例 |
|------|------|------|
| `T.tile.compare(dst, src0, src1, mode)` | 逐元素比较 | `T.tile.compare(c_ub, a_ub, b_ub, "EQ")` |
| `T.tile.select(dst, selMask, src0, src1, selMode)` | 条件选择 | `T.tile.select(c_ub, mask, a_ub, b_ub, "VSEL_CMPMASK_SPR")` |

**compare模式**：`"EQ"`、`"NE"`、`"GT"`、`"GE"`、`"LT"`、`"LE"`

### 4.6 归约操作

| 指令 | 功能 | 示例 |
|------|------|------|
| `T.reduce_sum(buffer, out, dim)` | 沿指定轴求和 | `T.reduce_sum(acc_s_ub, sumexp_ub, dim=-1)` |
| `T.reduce_max(buffer, out, dim)` | 沿指定轴求最大值 | `T.reduce_max(acc_s_ub, m_i, dim=-1)` |
| `T.reduce_min(buffer, out, dim)` | 沿指定轴求最小值 | `T.reduce_min(a_ub, b_ub, dim=-1)` |
| `T.reduce_max(buffer, out, dim, real_shape=[H,W])` | 带有效区域的最大值归约 | `T.reduce_max(in_ub, out_ub, dim=0, real_shape=[3,8])` |

- `dim=-1` 表示沿最后一个维度归约
- 输入为2D buffer，输出为1D buffer
- `real_shape` 参数：当 UB buffer 大于实际有效数据时，指定有效区域大小（如 buffer 为 (5,8)，但只有前3行有效，则 `real_shape=[3, 8]`）

### 4.7 位运算

| 指令 | 功能 |
|------|------|
| `T.tile.bitwise_and(dst, src0, src1)` | 逐元素按位与 |
| `T.tile.bitwise_or(dst, src0, src1)` | 逐元素按位或 |
| `T.tile.bitwise_not(dst, src0)` | 逐元素按位取反 |
| `T.tile.bitwise_xor(dst, src0, src1)` | 逐元素按位异或 |
| `T.tile.bitwise_lshift(dst, src0, scalar)` | 逐元素左移 |
| `T.tile.bitwise_rshift(dst, src0, scalar)` | 逐元素右移 |

## 五、同步指令

### 5.1 核内同步

#### T.barrier_all — 全局屏障

```python
T.barrier_all()
```

为所有流水线阶段插入一个屏障，确保所有之前提交的操作完成。

#### T.pipe_barrier — 流水线阶段屏障

```python
T.pipe_barrier("MTE3")
```

为特定的流水线阶段插入屏障。

#### T.sync_all — 计算单元全局同步

```python
T.sync_all()
```

在计算单元（AI Core）内执行全局同步。

### 5.2 核内流水线同步

#### T.set_flag / T.wait_flag — 流水线事件标志

```python
T.set_flag(src_pipe, dst_pipe, event_id)
T.wait_flag(src_pipe, dst_pipe, event_id)
```

**功能**：在流水线阶段之间进行事件同步。`set_flag` 由生产者设置标志，`wait_flag` 由消费者等待标志。

**参数**：
- `src_pipe`：源流水线阶段（生产者），如 `"mte2"`、`"mte1"`、`"m"`、`"fix"`
- `dst_pipe`：目的流水线阶段（消费者），如 `"v"`、`"mte3"`
- `event_id`：事件ID，用于区分不同的同步事件

**示例**：

```python
# MTE2完成数据加载后通知MTE1
T.set_flag("mte1", "mte2", 0)

# MTE1等待MTE2完成
T.wait_flag("mte1", "mte2", 0)
```

### 5.3 核间同步

#### T.set_cross_flag / T.wait_cross_flag — Cube与Vector核间同步

```python
T.set_cross_flag(pipe, flag)
T.wait_cross_flag(flag)
```

**功能**：Cube Core和Vector Core之间的跨核同步。Cube端设置标志，Vector端等待标志。

**参数**：
- `pipe`：发出设置的流水线阶段，如 `"FIX"`、`"MTE3"`、`"V"`
- `flag`：事件标志ID

**示例**（Flash Attention中Cube写workspace后通知Vector）：

```python
# Cube端（T.Scope("C")内）
T.copy(acc_s_l0c, workspace_1[cid, :, :])
T.set_cross_flag("FIX", 0)  # 通知Vector：数据已写入workspace

# Vector端（T.Scope("V")内）
T.wait_cross_flag(0)  # 等待Cube写入完成
T.copy(workspace_1[cid, ...], acc_s_ub)  # 安全读取workspace
```

## 六、调试指令

### T.printf — 设备端打印

```python
T.printf("fmt %s %d\n", "string", 0x123)
```

支持格式说明符：`%d`（十进制）、`%f`（浮点）、`%x`（十六进制）、`%s`（字符串）、`%p`（指针）。

### T.dump_tensor — 张量转储

```python
T.dump_tensor(tensor, desc, dump_size, shape_info=())
```

转储指定Tensor的内容，支持ub_buffer、l1_buffer、l0c_buffer和global_buffer。
