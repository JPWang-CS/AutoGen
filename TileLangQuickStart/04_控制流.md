# 04 控制流

TileLang-Ascend 提供了多种控制流结构，从基本的串行循环到高级的流水线并行和持久化调度，帮助开发者充分发挥昇腾NPU的硬件性能。

---

## 一、基本循环

### 1.1 T.serial — 串行循环

```python
for i in T.serial(N):
    # 循环体，i 从 0 到 N-1
    ...

# 带起止步长的串行循环
for i in T.serial(0, N, 2):
    # i = 0, 2, 4, ...
    ...
```

**功能**：串行执行循环体。等价于 Python 的 `for i in range(N):`。

**示例**：GEMM中沿K维度的循环

```python
loop_k = T.ceildiv(K, K_L1)
for k in T.serial(loop_k):
    T.copy(A[bx * block_M, k * K_L1], A_L1)
    T.copy(B[k * K_L1, by * block_N], B_L1)

    T.barrier_all()
    T.gemm_v0(A_L1, B_L1, C_L0, init=(k == 0))
    T.barrier_all()
```

---

### 1.2 T.unroll — 展开循环

```python
for k in T.unroll(K_TILE):
    acc += a[k] * b[k]
```

**功能**：完全展开循环，将循环体复制N份。适用于循环次数较少的场景，可以减少循环控制开销。

**适用场景**：
- 循环次数较少（通常 < 32）
- 需要消除循环控制开销

---

### 1.3 range — Python标准循环

```python
for h_i in range(block_M // 2):
    T.barrier_all()
    T.tile.sub(acc_s_ub[h_i, :], acc_s_ub[h_i, :], m_i[h_i])
    T.barrier_all()
```

**功能**：Python原生的 `range()` 循环。在TileLang内核中可用于顺序遍历，会被编译为串行循环。

---

## 二、并行循环

### 2.1 T.Parallel — 向量并行循环

```python
# 一维并行
for j in T.Parallel(block_N):
    c_ub[j] = a_ub[j] + b_ub[j]

# 二维并行
for (i, j) in T.Parallel(block_M // VEC_NUM, block_N):
    c_ub[i, j] = a_ub[i, j] + b_ub[i, j]
```

**功能**：表达Tile内元素级并行计算。在Ascend NPU上，`T.Parallel` 抽象了向量化的计算语义。

**两种编程范式**：

1. **T.Parallel + 符号API**（高级别，可移植）：

```python
for (i, j) in T.Parallel(block_M // VEC_NUM, block_N):
    b_ub[i, j] = T.exp(a_ub[i, j])
```

2. **T.tile.* 向量指令**（低级别，直接触发向量操作）：

```python
T.tile.exp(b_ub, a_ub)
```

两种方式等价，推荐根据场景选择：
- 简单表达式用 `T.Parallel`
- 复杂向量操作用 `T.tile.*`

**广播支持**：

```python
# 向量-标量运算
for j in T.Parallel(block_N):
    c_ub[j] = a_ub[j] + 1

# 行级广播
for (i, j) in T.Parallel(block_M // VEC_NUM, block_N):
    c_ub[i, j] = a_ub[i, j] * b_ub[i]  # b_ub是1D的
```

### NPU vs GPU T.Parallel 对比

**NPU (tilelang-ascend)**:
```python
# NPU: T.Parallel 表达向量语义，编译为 T.tile.* 或 VEC 指令
with T.Scope("V"):
    for (i, j) in T.Parallel(BM // 2, BN):
        c_ub[i, j] = a_ub[i, j] + b_ub[i, j]
```

**GPU (上游 TileLang)**:
```python
# GPU: T.Parallel 表达线程级并行，每个线程处理一个元素
for (i, j) in T.Parallel(BM, BN):
    c_shared[i, j] = a_shared[i, j] + b_shared[i, j]
```

| 差异点 | NPU | GPU |
|--------|-----|-----|
| 语义 | 向量指令（整个 Tile 并行） | 线程并行（每线程一元素） |
| 作用域 | 需在 `T.Scope("V")` 内 | 无需 |
| 执行单元 | Vector Core (AIV) | CUDA Cores |
| VEC_NUM | 需按 `VEC_NUM=2` 切分 | 无此概念 |
| 推荐替代 | `T.tile.*` 批量指令更直观 | `T.Parallel` 是主要方式 |

## 三、软件流水线

### 3.1 T.Pipelined — 流水线循环

```python
for k in T.Pipelined(T.ceildiv(K, K_L1), num_stages=2):
    T.copy(A[bx * block_M, k * K_L1], A_L1)
    T.copy(B[k * K_L1, by * block_N], B_L1)

    T.barrier_all()
    if k == 0:
        T.gemm_v0(A_L1, B_L1, C_L0, init=True)
    else:
        T.gemm_v0(A_L1, B_L1, C_L0)

    T.barrier_all()
```

**功能**：将循环迭代划分为多个阶段（stage），使数据加载和计算重叠执行，隐藏内存延迟。

**参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `extent` | int | 循环总次数 |
| `num_stages` | int | 流水线阶段数 |

**工作原理**（以 `num_stages=2`、`loop_k=4` 为例）：

```
无流水线：
  | copy_0 | gemm_0 | copy_1 | gemm_1 | copy_2 | gemm_2 | copy_3 | gemm_3 |

2级流水线（copy和gemm重叠）：
  | copy_0 | copy_1 | copy_2 + gemm_0 | copy_3 + gemm_1 | gemm_2 | gemm_3 |
```

**注意事项**：
- 核间流水线与核内流水线不能同时开启
- 使用核间流水线时，需开启 `TL_ASCEND_AUTO_CV_COMBINE` 和 `TL_ASCEND_AUTO_CV_SYNC`

### NPU vs GPU T.Pipelined 对比

**NPU (tilelang-ascend)**:
```python
# NPU: 核内流水线 — 重叠 MTE2(数据加载) 与 Cube(矩阵计算)
for k in T.Pipelined(loop_k, num_stages=2):
    T.copy(A[bx*BM, k*BK], A_L1)    # MTE2: GM -> L1
    T.copy(B[k*BK, by*BN], B_L1)    # MTE2: GM -> L1
    T.gemm_v0(A_L1, B_L1, C_L0)     # Cube: 矩阵乘

# NPU: 核间流水线 — 重叠 Cube 与 Vector (需 AUTO_CV_COMBINE)
for k in T.Pipelined(loop_k, num_stages=2, cross_interval=1):
    # Cube 写 workspace...
    # Vector 读 workspace...
```

**GPU (上游 TileLang)**:
```python
# GPU: 重叠 Shared Memory 加载与 Tensor Core 计算
for k in T.Pipelined(loop_k, num_stages=2):
    T.copy(A[bx*BM, k*BK], A_shared)   # GM -> Shared Memory
    T.copy(B[k*BK, by*BN], B_shared)   # GM -> Shared Memory
    T.gemm(A_shared, B_shared, C_frag)  # Tensor Core
```

| 差异点 | NPU | GPU |
|--------|-----|-----|
| 流水线类型 | 核内(核间可选) | 线程束级异步 |
| 重叠操作 | MTE2↔Cube, Cube↔Vector | Copy↔Compute |
| 多级流水线 | 支持 `num_stages=3` (MTE2-MTE1-M-FIX) | 通常 `num_stages=2` |
| 核间流水线 | `cross_interval` 控制 Cube↔Vector | 无此概念 |
| 同步要求 | 需 `pass_configs` 开启核间流水线 | 自动管理 |

### 3.2 T.Persistent — 持久化内核循环

```python
with T.Kernel(core_num, is_npu=True) as (cid, _):
    A_L1 = T.alloc_L1((block_M, K_L1), dtype)
    B_L1 = T.alloc_L1((K_L1, block_N), dtype)
    C_L0 = T.alloc_L0C((block_M, block_N), accum_dtype)

    with T.Scope("C"):
        for bx, by in T.Persistent(
            [T.ceildiv(M, block_M), T.ceildiv(N, block_N)],
            core_num, cid
        ):
            loop_k = T.ceildiv(K, K_L1)
            for k in T.serial(loop_k):
                T.copy(A[bx * block_M, k * K_L1], A_L1)
                T.copy(B[k * K_L1, by * block_N], B_L1)
                T.gemm_v0(A_L1, B_L1, C_L0, init=(k == 0))

            T.copy(C_L0, C[bx * block_M, by * block_N])
```

**功能**：将数据块在多个AI Core间进行缓存友好的调度，让加载的数据更容易命中缓存，提高L1 Buffer的复用率。

**参数**：

| 参数 | 说明 |
|------|------|
| `domain` | 数据块的2D网格大小，如 `[m_num, n_num]` |
| `wave_size` | 参与计算的AI Core数量 |
| `index` | 当前AI Core的编号（cid） |

**适用场景**：
- 数据块数量远大于AI Core数量
- 需要提高L1 Buffer的缓存命中率
- 相邻数据块共享部分数据（如K维度的共享）

## 四、通用控制流

### 4.1 条件语句

```python
if condition:
    ...
elif other_condition:
    ...
else:
    ...
```

TileLang支持标准的 `if/elif/else` 条件语句。条件应为TIR表达式。

**示例**：处理尾块（tail block）

```python
m_tail = M - m_num * block_M
n_tail = N - n_num * block_N

if bx < m_num and by < n_num:
    # 正常块
    ...
elif bx == m_num and by < n_num and m_tail > 0:
    # M维度的尾块
    ...
elif bx < m_num and by == n_num and n_tail > 0:
    # N维度的尾块
    ...
```

### 4.2 T.ceildiv — 向上取整除法

```python
loop_k = T.ceildiv(K, K_L1)  # 等价于 (K + K_L1 - 1) // K_L1
```

常用于计算需要的循环次数，确保覆盖所有数据。

### 4.3 T.And — 逻辑与

```python
T.mma(A_L0[kk % S2], B_L0[kk % S2], C_L0, init=T.And(k == 0, kk == 0))
```

**功能**：TIR级别的逻辑与操作，用于组合多个条件。仅在首次迭代时清零累加器。

### 4.4 while 循环

```python
i = 0
while i < N:
    ...
    if done:
        break
    i += 1
```

**注意**：循环条件需要是TIR表达式。TileLang如果检测出死循环会编译报错。

### 4.5 break 和 continue

```python
for i in T.serial(N):
    if skip_condition:
        continue  # 跳过当前迭代
    if stop_condition:
        break     # 终止循环
    ...
```

在 `T.serial`、`T.unroll`、`T.Parallel`、`while` 循环中均可使用 `break` 和 `continue`。

## 五、控制流选择指南

| 控制流 | 适用场景 | 性能特点 |
|--------|---------|---------|
| `T.serial` | 串行迭代（如K维度循环） | 无并行开销 |
| `T.unroll` | 小范围循环（< 32） | 消除循环开销 |
| `T.Parallel` | Tile内逐元素操作 | 向量化并行 |
| `T.Pipelined` | 计算密集型循环（copy+gemm重叠） | 隐藏内存延迟 |
| `T.Persistent` | 多数据块多核调度 | 提高缓存命中率 |
| `while` | 动态终止条件 | 灵活但需注意性能 |

## 六、完整示例：GEMM中的控制流组合

```python
@tilelang.jit(out_idx=[-1])
def matmul(M, N, K, block_M, block_N, K_L1, dtype="float16", accum_dtype="float"):
    m_num = M // block_M
    n_num = N // block_N
    core_num = 24

    @T.prim_func
    def main(
        A: T.Tensor((M, K), dtype),
        B: T.Tensor((K, N), dtype),
        C: T.Tensor((M, N), dtype),
    ):
        with T.Kernel(core_num, is_npu=True) as (cid, _):
            A_L1 = T.alloc_L1((block_M, K_L1), dtype)
            B_L1 = T.alloc_L1((K_L1, block_N), dtype)
            C_L0 = T.alloc_L0C((block_M, block_N), accum_dtype)

            with T.Scope("C"):
                # T.Persistent：多块持久化调度
                for bx, by in T.Persistent(
                    [T.ceildiv(M, block_M), T.ceildiv(N, block_N)],
                    core_num, cid
                ):
                    # T.ceildiv：计算循环次数
                    loop_k = T.ceildiv(K, K_L1)

                    # T.serial：沿K维度串行循环
                    for k in T.serial(loop_k):
                        T.copy(A[bx * block_M, k * K_L1], A_L1)
                        T.copy(B[k * K_L1, by * block_N], B_L1)

                        T.barrier_all()
                        # 条件判断：首次清零，后续累加
                        T.gemm_v0(A_L1, B_L1, C_L0, init=(k == 0))
                        T.barrier_all()

                    T.copy(C_L0, C[bx * block_M, by * block_N])

    return main
```
