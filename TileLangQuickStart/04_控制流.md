# 04 控制流

TileLang 提供了多种控制流结构，从基本的串行循环到高级的软件流水线，帮助开发者充分发挥硬件性能。

---

## 一、基本循环

### 1.1 T.serial — 串行循环

```python
for i in T.serial(N):
    # 循环体
    ...
```

**功能**：串行执行循环体 N 次，`i` 从 0 到 N-1。

**等价于 Python 的**：
```python
for i in range(N):
    ...
```

**示例**：

```python
# 逐元素操作
for i in T.serial(BM):
    for j in T.serial(BN):
        C[i, j] = A[i, j] + B[i, j]
```

---

### 1.2 T.unroll — 展开循环

```python
for i in T.unroll(N):
    # 循环体
    ...
```

**功能**：完全展开循环，将循环体复制 N 份。适用于循环次数较少的场景，可以减少循环控制开销。

**适用场景**：
- 循环次数较少（通常 < 32）
- 需要消除循环控制开销
- 编译器可以更好地优化展开后的代码

**示例**：

```python
# 展开 4 次循环，减少分支开销
for k in T.unroll(4):
    T.gemm(A_s[k], B_s[k], C_f)
```

---

## 二、并行循环

### 2.1 T.Parallel — 并行循环

```python
for i in T.Parallel(N):
    # 每次迭代独立执行
    ...
```

**功能**：将循环迭代映射到线程。每次迭代可以独立、并行地执行。

**与 T.serial 的区别**：
- `T.serial`：循环体在时间上串行执行
- `T.Parallel`：循环体在空间上并行执行（由不同线程同时执行）

**约束**：循环迭代之间不能有数据依赖。

**示例**：

```python
# 并行初始化多个缓冲区
for i in T.Parallel(3):
    # i=0,1,2 分别由不同线程处理
    buffer[i] = 0.0
```

**可选参数**：

| 参数 | 说明 |
|------|------|
| `coalesced_width` | 控制内存合并访问宽度（用于向量化检查） |
| `loop_layout` | 接受一个 `T.Fragment`，标注整个嵌套并行循环的布局 |

> **提示**：`loop_layout` 注解仅附加到最外层循环，且要求 `InputDim` 等于嵌套并行维度的数量。

## 三、软件流水线

### 3.1 T.Pipelined — 软件流水线循环

```python
for ko in T.Pipelined(T.ceildiv(K, BK), num_stages=3):
    T.copy(A[by * BM, ko * BK], A_shared)  # 阶段 N: 加载数据
    T.copy(B[ko * BK, bx * BN], B_shared)
    T.gemm(A_shared, B_shared, C_frag)       # 阶段 N: 计算
```

**功能**：将循环迭代划分为多个阶段（stage），使数据加载和计算重叠执行，从而隐藏内存延迟。

**参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `extent` | int | 循环总次数 |
| `num_stages` | int | 流水线阶段数（默认 1，即无流水线） |

**工作原理**：

```
无流水线（num_stages=1）:
  时间 →  |  Load 0 | Compute 0 |  Load 1 | Compute 1 |  Load 2 | Compute 2 |

2 级流水线（num_stages=2）:
  时间 →  |  Load 0 | Load 1 + Compute 0 | Load 2 + Compute 1 | Compute 2 |

3 级流水线（num_stages=3）:
  时间 →  |  Load 0 | Load 1 + ... | Load 2 + Compute 0 | Compute 1 | ...
```

**特点**：
- 自动在 Shared Memory 中分配双缓冲/多缓冲
- 自动使用 `cp.async` 进行异步数据加载（如果硬件支持）
- 需要额外的 Shared Memory 用于缓冲区

**GEMM 中的典型使用**：

```python
@T.prim_func
def gemm_pipelined(A: T.Tensor((M, K), "float16"),
                   B: T.Tensor((K, N), "float16"),
                   C: T.Tensor((M, N), "float16")):
    with T.Kernel(T.ceildiv(N, BN), T.ceildiv(M, BM), threads=128) as (bx, by):
        A_s = T.alloc_shared((BM, BK), "float16")
        B_s = T.alloc_shared((BK, BN), "float16")
        C_f = T.alloc_fragment((BM, BN), "float32")

        T.clear(C_f)

        # 软件流水线：加载 K/BK 次，每次加载 BK 列
        for ko in T.Pipelined(T.ceildiv(K, BK), num_stages=3):
            T.copy(A[by * BM, ko * BK], A_s)
            T.copy(B[ko * BK, bx * BN], B_s)
            T.gemm(A_s, B_s, C_f)

        T.copy(C_f, C[by * BM, bx * BN])
```

### 3.2 T.Persistent — 持久化内核循环

```python
with T.Persistent(domain, wave_size, index, group_size=...) as (bx, by):
    # 持久化循环体
    for _ in T.serial(num_iter):
        # 从工作队列获取任务
        # 执行计算
        ...
```

**功能**：用于实现持久化内核（Persistent Kernel），内核在 GPU 上持续运行，从工作队列中动态获取任务。

**参数**：

| 参数 | 说明 |
|------|------|
| `domain` | 总任务域 |
| `wave_size` | 每波次的 block 数量 |
| `index` | 当前 block 的索引 |
| `group_size` | 分组大小（可选） |

**适用场景**：
- Flash Attention 等需要动态调度的算法
- 减少内核启动开销
- 需要 Block 级别的动态任务调度

> **注意**：`T.Persistent` 是高级构造，TileLang 在后续编译 Pass 中进行专门处理，通常由专业模板使用。

## 四、通用控制流

### 4.1 while 循环

```python
while condition:
    # 循环体
    ...
```

**功能**：条件循环，当条件为真时持续执行。

**示例**：

```python
# 迭代求解
x = T.alloc_fragment((N,), "float32")
error = 1.0
while error > 1e-6:
    # 更新 x
    ...
    # 计算新的误差
    error = compute_error(x)
```

### 4.2 break 和 continue

```python
for i in T.serial(N):
    if skip_condition:
        continue  # 跳过当前迭代
    if stop_condition:
        break     # 终止循环
    # 正常处理
    ...
```

**功能**：
- `continue`：跳过当前迭代，进入下一次迭代
- `break`：立即终止循环

### 4.3 条件语句

```python
if condition:
    # 条件为真时执行
    ...
else:
    # 条件为假时执行
    ...
```

TileLang 支持标准的 if/elif/else 条件语句，可以在 Kernel 内使用。

### 多条件谓词

使用 `T.any_of` / `T.all_of` 表达多条件判断：

```python
if T.all_of(i < M, j < N):
    C[i, j] = A[i, j] + B[i, j]

if T.any_of(x < 0, x > 100):
    # 处理越界
    ...
```

### 边界处理

TileLang 的 **LegalizeSafeMemoryAccess** 编译 Pass 会自动在可能越界的访问处插入保护代码，并在可证明安全时移除它们。对于简单的边界处理，通常可以省略显式的 `if` 检查：

```python
for i, j in T.Parallel(M, N):
    gi = by * BM + i
    gj = bx * BN + j
    if T.all_of(gi < M, gj < N):  # 在多数情况下可省略
        C[gi, gj] = A[gi, gj] + B[gi, gj]
```

## 五、控制流选择指南

| 控制流 | 适用场景 | 性能特点 |
|--------|---------|---------|
| `T.serial` | 简单串行循环 | 无并行开销 |
| `T.unroll` | 小范围循环（< 32） | 消除循环开销 |
| `T.Parallel` | 独立并行任务 | 充分利用线程 |
| `T.Pipelined` | 计算密集型循环 | 隐藏内存延迟 |
| `T.Persistent` | 动态任务调度 | 减少内核启动开销 |
| `while` | 动态终止条件 | 灵活但需注意性能 |
