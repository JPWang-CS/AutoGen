## 1. T.alloc_var -- 标量变量分配

用于在 kernel 中分配标量变量（标志位、计数器、临时标量）。

```python
# 基本用法
flag = T.alloc_var("bool", init=False)        # 布尔标志位
counter = T.alloc_var("int32", init=1)         # 整数计数器
value = T.alloc_var("float32", init=0.0)       # 浮点临时变量

# 变量间初始化
a = T.alloc_var("int32", init=1)
b = T.alloc_var("int32", init=a)               # 用 a 的值初始化 b

# 在条件逻辑中使用
flag = T.alloc_var("bool", init=False)
if flag:
    a = 2
else:
    a = a + 1

# 指定内存作用域
var1 = T.alloc_var("int32", "local.var", init=1)
var2 = T.alloc_var("int32", init=1, scope="local.var")
```

**参数**:
- `dtype`: 数据类型，如 `"bool"`, `"int32"`, `"float32"`
- `init`: 初始值，可选常量、表达式或变量
- `scope`: 内存作用域，默认 `"local.var"`

## 3. T.copy 支持的数据搬运路径

| src | dst | 说明 |
|-----|-----|------|
| GM | L1 | Global Memory 到 L1 Buffer (Cube 数据加载) |
| L1 | L0A | L1 到 L0A (Cube 左矩阵输入) |
| L1 | L0B | L1 到 L0B (Cube 右矩阵输入) |
| L0C | GM | L0C 到 Global Memory (Cube 结果写回) |
| GM | UB | Global Memory 到 Unified Buffer (Vector 数据加载) |
| UB | GM | Unified Buffer 到 Global Memory (Vector 结果写回) |
| UB | UB | UB 到 UB 拷贝 |
| UB | L1 | UB 到 L1 (Vector 到 Cube 数据传递) |

**零填充**: 当检测到加载越界时，自动用零填充。

## 4. T.Pipelined 详细用法

### 4.1 核内流水线 (Intra-core)

重叠单个核心内的内存访问和计算:

```python
for k in T.Pipelined(loop_k, num_stages=2):
    T.copy(A[bx * block_M, k * block_K], A_L1)
    T.copy(B[k * block_K, by * block_N], B_L1)
    T.barrier_all()
    if k == 0:
        T.gemm_v0(A_L1, B_L1, C_L0, init=True)
    else:
        T.gemm_v0(A_L1, B_L1, C_L0)
    T.barrier_all()
```

执行时序 (num_stages=2, loop_k=4):
- Prefetch 阶段: copy_A_0, copy_B_0, copy_A_1, copy_B_1
- Main body 阶段: copy+gemm 交替执行
- Epilogue 阶段: gemm_2, gemm_3

### 4.2 核间流水线 (Inter-core)

重叠 Cube 和 Vector 核心的操作:

```python
for k in T.Pipelined(T.ceildiv(seq_len, block_N), num_stages=2):
    # Cube: 写入 workspace
    T.copy(K[bz, by, k * block_N:(k+1) * block_N, :], k_l1)
    T.gemm_v0(q_l1, k_l1, acc_s_l0c, transpose_B=True, init=True)
    T.copy(acc_s_l0c, workspace_1[cid, :, :])

    # Vector: 从 workspace 读取
    T.copy(workspace_1[cid, vid*block_M//2:..., :], acc_s_ub_)
    T.tile.add(acc_s_ub, acc_s_ub, acc_s_ub_)
```

**必须开启**:
```python
pass_configs = {
    tilelang.PassConfigKey.TL_ASCEND_AUTO_CV_COMBINE: True,
    tilelang.PassConfigKey.TL_ASCEND_AUTO_CV_SYNC: True,
}
```

### 4.3 cross_interval 参数 (仅核间)

控制核间同步频率:

```python
# 默认: 每次迭代同步
for k in T.Pipelined(num_iters, num_stages=4, cross_interval=1):

# 每2次迭代同步一次
for k in T.Pipelined(num_iters, num_stages=4, cross_interval=2):
```

### 4.4 约束

- **核内流水线和核间流水线不能嵌套**。使用 flat pattern。
- 一个程序内不支持多个核间流水线。
- 使用核间流水线时必须开启 AUTO_CV_COMBINE 和 AUTO_CV_SYNC。

## 5. T.Persistent 详细用法

优化数据块在 AI Core 间的调度策略，提高缓存命中率:

```python
with T.Kernel(m_num * n_num, is_npu=True) as (cid, _):
    A_L1 = T.alloc_shared((block_M, K_L1), dtype)
    B_L1 = T.alloc_shared((K_L1, block_N), dtype)
    C_L0 = T.alloc_fragment((block_M, block_N), accum_dtype)

    for bx, by in T.Persistent(
        [T.ceildiv(M, block_M), T.ceildiv(N, block_N)],
        core_num, cid):
        loop_k = T.ceildiv(K, K_L1)
        for k in T.serial(loop_k):
            T.copy(A[bx * block_M, k * K_L1], A_L1)
            T.copy(B[k * K_L1, by * block_N], B_L1)
            T.gemm_v0(A_L1, B_L1, C_L0, init=(k == 0))
        T.copy(C_L0, C[bx * block_M, by * block_N])
```

## 6. Developer 模式详细说明

### 6.1 Developer 模式的核心理念

Developer 模式提供 Tile Library 接口，用户无需关心底层硬件细节:
- 使用 `T.alloc_shared()` 替代 `T.alloc_L1()` / `T.alloc_ub()` -- 编译器自动映射
- 使用 `T.alloc_fragment()` 替代 `T.alloc_L0C()` -- 编译器自动识别
- 使用 `T.Parallel` + 符号 API 或 `T.tile.*` 进行向量计算
- 通过 `pass_configs` 自动插入同步和内存规划

### 6.2 自动映射规则

```
T.alloc_shared() -> L1 Buffer (当上下文为 Cube 操作时)
                  -> UB       (当上下文为 Vector 操作时)

T.alloc_fragment() -> L0A/L0B/L0C (编译器根据 gemm 输入输出自动判断)
```

### 6.3 Developer 模式 GEMM 示例

来源: `tilelang-ascend/examples/developer_mode/gemm_developer.py`

```python
pass_configs = {
    tilelang.PassConfigKey.TL_ASCEND_AUTO_CV_COMBINE: True,
    tilelang.PassConfigKey.TL_ASCEND_AUTO_SYNC: True,
    tilelang.PassConfigKey.TL_ASCEND_MEMORY_PLANNING: True,
    tilelang.PassConfigKey.TL_ASCEND_AUTO_CV_SYNC: True,
}

@tilelang.jit(out_idx=[-1], pass_configs=pass_configs)
def matmul(M, N, K, block_M, block_N, K_L1, dtype="float16", accum_dtype="float"):
    m_num = M // block_M
    n_num = N // block_N

    @T.prim_func
    def main(A, B, C):
        with T.Kernel(m_num * n_num, is_npu=True) as (cid, _):
            bx = cid // n_num
            by = cid % n_num
            A_L1 = T.alloc_shared((block_M, K_L1), dtype)     # 自动映射到 L1
            B_L1 = T.alloc_shared((K_L1, block_N), dtype)     # 自动映射到 L1
            C_L0 = T.alloc_fragment((block_M, block_N), accum_dtype)  # 自动映射到 L0C

            loop_k = T.ceildiv(K, K_L1)
            for k in T.serial(loop_k):
                T.copy(A[bx * block_M, k * K_L1], A_L1)
                T.copy(B[k * K_L1, by * block_N], B_L1)
                T.gemm_v0(A_L1, B_L1, C_L0, init=(k == 0))   # 无需 T.Scope

            T.copy(C_L0, C[bx * block_M, by * block_N])
    return main
```

**关键区别**:
- 无需 `T.Scope("C")` -- 编译器自动识别
- 无需 `T.barrier_all()` -- pass_configs 自动插入
- 使用 `T.alloc_shared` / `T.alloc_fragment` 而非 `T.alloc_L1` / `T.alloc_L0C`

## 9. Workspace 自动分配

使用 `workspace_idx` 参数声明 workspace 参数位置:

```python
@tilelang.jit(out_idx=[3], workspace_idx=[4, 5, 6, 7])
def sparse_attention_fwd(...):
    @T.prim_func
    def main(
        Q: T.Tensor(q_shape, dtype),          # 输入
        KV: T.Tensor(kv_shape, dtype),         # 输入
        Indices: T.Tensor(indices_shape, ...), # 输入
        Output: T.Tensor(o_shape, dtype),      # 输出 (out_idx=3)
        workspace_1: T.Tensor([...], dtype),   # 自动管理 (workspace_idx=4)
        workspace_2: T.Tensor([...], dtype),   # 自动管理 (workspace_idx=5)
        workspace_3: T.Tensor([...], accum_dtype),
        workspace_4: T.Tensor([...], dtype),
    ):
        # 使用 workspace 作为临时缓冲区
        T.copy(result, workspace_1[cid, :, :])
        ...
```

用户调用时只需传入输入参数，workspace 由框架自动管理:

```python
output = sparse_attention_op(q, kv, indices)
```

> **限制**: 当前仅 Cython 后端支持。
