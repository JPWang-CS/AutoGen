# 08 实战示例：注意力机制

本章介绍如何使用 TileLang 实现 Flash Attention 风格的注意力机制内核，这是 TileLang 最典型的应用场景之一。

---

## 一、注意力机制原理

标准注意力计算公式：

```
Attention(Q, K, V) = softmax(Q @ K^T / sqrt(d)) @ V
```

其中：
- `Q`：查询矩阵，形状 `(B, H, L, d)`
- `K`：键矩阵，形状 `(B, H, L, d)`
- `V`：值矩阵，形状 `(B, H, L, d)`
- `d`：注意力头维度
- `L`：序列长度

### Flash Attention 的核心思想

标准注意力需要将完整的 `L × L` 注意力矩阵存入全局内存（O(L^2) 显存），而 Flash Attention 通过 **在线 Softmax（Online Softmax）** 技巧，将计算分块进行，只需 O(L) 的额外显存。

**在线 Softmax 算法**：

```
对每个 Q 块 (大小 Br × d):
    初始化: O = 0, m = -inf, l = 0

    对每个 KV 块 (大小 Bc × d):
        S = Q_block @ K_block^T          # 计算注意力分数
        m_new = max(m, rowmax(S))         # 更新行最大值
        P = exp(S - m_new)                # 数值稳定
        l_new = l * exp(m - m_new) + rowsum(P)  # 更新归一化因子
        O = O * (l * exp(m - m_new) / l_new) + P @ V_block / l_new  # 更新输出

    O = O / l                             # 最终归一化
```

## 二、TileLang 实现

### 2.1 简化版 Flash Attention

```python
import tilelang
import tilelang.language as T
import torch

def flash_attention(
    batch: int, heads: int, seq_len: int, dim: int,
    block_M: int = 64, block_N: int = 64,
    dtype: str = "float16", accum_dtype: str = "float32",
):
    BM, BN = block_M, block_N
    scale = (1.0 / (dim ** 0.5))

    @T.prim_func
    def main(
        Q: T.Tensor((batch, heads, seq_len, dim), dtype),
        K: T.Tensor((batch, heads, seq_len, dim), dtype),
        V: T.Tensor((batch, heads, seq_len, dim), dtype),
        Output: T.Tensor((batch, heads, seq_len, dim), dtype),
    ):
        # Grid: (seq_len/BN, heads, batch)
        with T.Kernel(
            T.ceildiv(seq_len, BM), heads, batch, threads=128
        ) as (bx, by, bz):
            # 分配共享内存
            Q_shared = T.alloc_shared((BM, dim), dtype)
            K_shared = T.alloc_shared((BN, dim), dtype)
            V_shared = T.alloc_shared((BN, dim), dtype)

            # Fragment（寄存器）缓冲区
            S_frag = T.alloc_fragment((BM, BN), accum_dtype)     # 注意力分数
            O_frag = T.alloc_fragment((BM, dim), accum_dtype)    # 输出累加器
            m_frag = T.alloc_fragment((BM, 1), accum_dtype)      # 行最大值
            l_frag = T.alloc_fragment((BM, 1), accum_dtype)      # 行归一化因子

            # 初始化
            T.fill(m_frag, T.float32(-1e9))  # -inf
            T.fill(l_frag, T.float32(0.0))   # 0
            T.clear(O_frag)                    # 0

            # 加载 Q 块（整个序列只需要加载一次）
            T.copy(Q[bz, by, bx * BM, 0], Q_shared)

            # 沿 K/V 序列维度分块循环
            for ko in T.serial(T.ceildiv(seq_len, BN)):
                # 加载 K 和 V 的当前块
                T.copy(K[bz, by, ko * BN, 0], K_shared)
                T.copy(V[bz, by, ko * BN, 0], V_shared)

                # 计算 S = Q @ K^T * scale
                T.gemm(Q_shared, T.transposed(K_shared), S_frag)

                # 应用 scale
                for i in T.serial(BM):
                    for j in T.serial(BN):
                        S_frag[i, j] = S_frag[i, j] * scale

                # 更新行最大值
                m_new = T.alloc_fragment((BM, 1), accum_dtype)
                for i in T.serial(BM):
                    m_new[i, 0] = m_frag[i, 0]
                    for j in T.serial(BN):
                        m_new[i, 0] = T.max(m_new[i, 0], S_frag[i, j])

                # 计算修正因子
                correction = T.alloc_fragment((BM, 1), accum_dtype)
                for i in T.serial(BM):
                    correction[i, 0] = T.exp(m_frag[i, 0] - m_new[i, 0])

                # 修正之前的累加值
                for i in T.serial(BM):
                    for j in T.serial(dim):
                        O_frag[i, j] = O_frag[i, j] * correction[i, 0]
                    l_frag[i, 0] = l_frag[i, 0] * correction[i, 0]

                # 计算 softmax 分子
                P_frag = T.alloc_fragment((BM, BN), accum_dtype)
                for i in T.serial(BM):
                    for j in T.serial(BN):
                        P_frag[i, j] = T.exp(S_frag[i, j] - m_new[i, 0])

                # 更新归一化因子
                for i in T.serial(BM):
                    for j in T.serial(BN):
                        l_frag[i, 0] = l_frag[i, 0] + P_frag[i, j]

                # P @ V 累加到输出
                T.gemm(P_frag, V_shared, O_frag)

                # 更新 m
                for i in T.serial(BM):
                    m_frag[i, 0] = m_new[i, 0]

            # 最终归一化
            for i in T.serial(BM):
                for j in T.serial(dim):
                    O_frag[i, j] = O_frag[i, j] / l_frag[i, 0]

            # 写回结果
            T.copy(O_frag, Output[bz, by, bx * BM, 0])

    return main
```

### 2.2 使用软件流水线优化的版本

```python
@tilelang.jit(out_idx=[3])
def flash_attention_pipelined(
    batch: int, heads: int, seq_len: int, dim: int,
    block_M: int = 64, block_N: int = 64,
    num_stages: int = 2, num_threads: int = 128,
    dtype: str = "float16", accum_dtype: str = "float32",
):
    BM, BN = block_M, block_N
    scale = (1.0 / (dim ** 0.5))

    @T.prim_func
    def main(
        Q: T.Tensor((batch, heads, seq_len, dim), dtype),
        K: T.Tensor((batch, heads, seq_len, dim), dtype),
        V: T.Tensor((batch, heads, seq_len, dim), dtype),
        Output: T.Tensor((batch, heads, seq_len, dim), dtype),
    ):
        with T.Kernel(
            T.ceildiv(seq_len, BM), heads, batch, threads=num_threads
        ) as (bx, by, bz):
            Q_shared = T.alloc_shared((BM, dim), dtype)
            K_shared = T.alloc_shared((BN, dim), dtype)
            V_shared = T.alloc_shared((BN, dim), dtype)

            S_frag = T.alloc_fragment((BM, BN), accum_dtype)
            O_frag = T.alloc_fragment((BM, dim), accum_dtype)
            m_frag = T.alloc_fragment((BM, 1), accum_dtype)
            l_frag = T.alloc_fragment((BM, 1), accum_dtype)

            T.fill(m_frag, T.float32(-1e9))
            T.fill(l_frag, T.float32(0.0))
            T.clear(O_frag)

            T.copy(Q[bz, by, bx * BM, 0], Q_shared)

                # S = Q @ K^T
                T.gemm(Q_shared, T.transposed(K_shared), S_frag)

                # 在线 Softmax 更新逻辑（同上）
                # ...

            T.copy(O_frag, Output[bz, by, bx * BM, 0])

    return main
```

## 四、Flash Attention 优化技巧

### 4.1 分块大小选择

| 分块大小 | Shared Memory 使用 | 适用场景 |
|---------|-------------------|---------|
| BM=64, BN=64 | 较少 | 长序列、头维度大 |
| BM=128, BN=128 | 中等 | 通用场景 |
| BM=64, BN=128 | 适中 | K/V 循环优化 |

### 4.2 序列并行

对于不同批次和注意力头，TileLang 会自动映射到不同的 Thread Block：

```python
# Grid 维度: (seq_len/BM, heads, batch)
with T.Kernel(
    T.ceildiv(seq_len, BM), heads, batch, threads=num_threads
) as (bx, by, bz):
    # bx: 序列分块索引
    # by: 注意力头索引
    # bz: 批次索引
    ...
```

### 4.3 因果注意力（Causal Attention）

对于自回归模型，需要应用因果掩码：

```python
# 在 S = Q @ K^T 之后添加掩码
for i in T.serial(BM):
    for j in T.serial(BN):
        # 计算全局位置
        qi = bx * BM + i
        kj = ko * BN + j
        if kj > qi:
            S_frag[i, j] = T.float32(-1e9)  # 掩码位置设为 -inf
```

## 六、TileLang 中注意力机制的变体

TileLang 的灵活性使其可以实现多种注意力变体：

| 变体 | 说明 | TileLang 关键特性 |
|------|------|-------------------|
| 标准 Flash Attention | 高效注意力计算 | `T.gemm`, 在线 Softmax |
| 因果注意力 | 自回归模型 | 条件语句 + 掩码 |
| GQA (Grouped Query Attention) | 共享 KV 头 | Grid 维度调整 |
| MLA (Multi-head Latent Attention) | DeepSeek 风格 | 自定义 KV 压缩 |
| 稀疏注意力 | Block-sparse | 条件跳过 |
| Sliding Window Attention | 局部注意力 | 循环范围限制 |

## 总结

本章展示了如何使用 TileLang 实现高性能的注意力机制内核。关键要点：

1. **在线 Softmax** 是 Flash Attention 的核心算法
2. **分块策略**（block_M, block_N）直接影响性能和 Shared Memory 使用
3. **T.gemm** 利用 Tensor Core 加速矩阵乘法
4. TileLang 的灵活控制流支持实现各种注意力变体
