# 06 自动调优

TileLang 内置了强大的自动调优（Autotuning）功能，帮助开发者自动搜索最优的内核配置参数，无需手动调参。

---

## 一、概述

GPU 内核的性能高度依赖于配置参数（如 Block 大小、流水线级数、线程数等）。手动调优耗时且困难，TileLang 提供了两种自动调优方式：

1. **装饰器模式**：使用 `@tilelang.autotune` 装饰器，简洁直观
2. **编程模式**：使用 `AutoTuner` 类，灵活可编程

---

## 二、装饰器模式

### 2.1 基本用法

```python
import tilelang
import tilelang.language as T

# 定义配置空间
matmul_configs = [
    {"block_M": 64,  "block_N": 64,  "block_K": 32, "threads": 64,  "num_stages": 2},
    {"block_M": 128, "block_N": 128, "block_K": 32, "threads": 128, "num_stages": 2},
    {"block_M": 128, "block_N": 128, "block_K": 64, "threads": 128, "num_stages": 3},
    {"block_M": 128, "block_N": 256, "block_K": 32, "threads": 128, "num_stages": 3},
]

# 使用 @tilelang.autotune 装饰器
@tilelang.autotune(
    configs=matmul_configs,    # 配置列表
    warmup=25,                  # 预热迭代次数
    rep=100,                    # 测量迭代次数
    timeout=60,                 # 超时时间（秒）
)
@tilelang.jit(out_idx=[-1])    # 必须放在 autotune 下方
def matmul(
    M, N, K,
    block_M=128, block_N=128, block_K=32,
    threads=128, num_stages=3,
    dtype="float16", accum_dtype="float32"
):
    @T.prim_func
    def kernel(
        A: T.Tensor((M, K), dtype),
        B: T.Tensor((K, N), dtype),
        C: T.Tensor((M, N), accum_dtype),
    ):
        with T.Kernel(T.ceildiv(N, block_N), T.ceildiv(M, block_M), threads=threads) as (bx, by):
            A_s = T.alloc_shared((block_M, block_K), dtype)
            B_s = T.alloc_shared((block_K, block_N), dtype)
            C_f = T.alloc_fragment((block_M, block_N), accum_dtype)
            T.clear(C_f)
            for ko in T.Pipelined(T.ceildiv(K, block_K), num_stages=num_stages):
                T.copy(A[by * block_M, ko * block_K], A_s)
                T.copy(B[ko * block_K, bx * block_N], B_s)
                T.gemm(A_s, B_s, C_f)
            T.copy(C_f, C[by * block_M, bx * block_N])
    return kernel

# 自动调优并执行
import torch
M, N, K = 1024, 1024, 1024
A = torch.randn(M, K, device="cuda", dtype=torch.float16)
B = torch.randn(K, N, device="cuda", dtype=torch.float16)

# 首次调用会自动搜索最优配置
C = matmul(A, B)
```

### 2.2 @tilelang.autotune 参数详解

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `configs` | list[dict] | 必填 | 配置空间列表，每个字典是一组参数值 |
| `warmup` | int | 25 | 预热迭代次数（不计入性能测量） |
| `rep` | int | 100 | 性能测量迭代次数 |
| `timeout` | float | 60 | 单个配置的最大评估时间（秒） |
| `target` | str | "cuda" | 编译目标 |

## 三、编程模式

使用 `AutoTuner` 类实现更灵活的调优控制：

```python
from tilelang.autotune import AutoTuner

# 创建调优器
tuner = AutoTuner(
    kernel_func=matmul_func,     # 编译函数
    configs=matmul_configs,       # 配置空间
    warmup=25,
    rep=100,
    timeout=60,
)

# 指定输入张量（用于编译和性能测试）
import torch
M, N, K = 1024, 1024, 1024
inputs = [
    torch.randn(M, K, device="cuda", dtype=torch.float16),
    torch.randn(K, N, device="cuda", dtype=torch.float16),
    torch.randn(M, N, device="cuda", dtype=torch.float32),
]

# 运行调优
best_config = tuner.tune(inputs)

print(f"最优配置: {best_config}")
```

### 3.1 AutoTuner 接口详解

| 方法 | 说明 |
|------|------|
| `tune(inputs)` | 运行自动调优，返回最优配置字典 |
| `tune_and_apply(inputs)` | 调优并返回使用最优配置编译的内核 |

## 四、配置空间定义

### 4.1 手动枚举

```python
configs = [
    {"block_M": 64,  "block_N": 64,  "block_K": 32, "threads": 64},
    {"block_M": 128, "block_N": 128, "block_K": 32, "threads": 128},
    {"block_M": 128, "block_N": 256, "block_K": 64, "threads": 256},
]
```

### 4.2 使用 itertools 生成

```python
import itertools

block_sizes = [64, 128, 256]
k_sizes = [32, 64]
stages = [2, 3]
thread_counts = [128, 256]

configs = []
for bm, bk, ns, nt in itertools.product(block_sizes, k_sizes, stages, thread_counts):
    configs.append({
        "block_M": bm, "block_N": bm,
        "block_K": bk,
        "num_stages": ns,
        "threads": nt,
    })
```

### 4.3 条件过滤

```python
# 过滤掉不合理的配置
valid_configs = [
    c for c in all_configs
    if c["block_M"] * c["block_K"] <= 4096  # Shared Memory 限制
    and c["threads"] % 32 == 0               # Warp 对齐
]
```

## 五、提供输入数据

自动调优需要输入张量来编译和测试内核。有两种方式提供：

### 5.1 在调优函数中自动生成

`@tilelang.autotune` 装饰器会在首次调用时使用传入的张量进行调优。

### 5.2 显式提供参考输入

```python
@tilelang.autotune(
    configs=configs,
    warmup=25,
    rep=100,
)
@tilelang.jit(out_idx=[-1])
def my_kernel(M, N, block_M=128, block_N=128, dtype="float16"):
    ...
    return kernel

# 调用时传入真实数据
A = torch.randn(M, K, device="cuda", dtype=torch.float16)
B = torch.randn(K, N, device="cuda", dtype=torch.float16)
C = my_kernel(A, B)  # 首次调用触发调优
```

## 九、编程模式（AutoTuner 类）

除了装饰器模式，TileLang 还提供了编程式的 `AutoTuner` 类，实现更灵活的调优控制：

```python
from tilelang.autotuner import AutoTuner

# 创建调优器
tuner = AutoTuner.from_kernel(
    kernel_factory=matmul(M, N, K),  # 已编译的 kernel
    configs=matmul_configs(M, N, K),
)

tuner.set_profile_args(
    warmup=25,
    rep=100,
    timeout=60,
    supply_type=tilelang.TensorSupplyType.Auto,  # 或提供 supply_prog/ref_prog
    ref_prog=lambda A, B, C: torch.allclose(C, (A @ B).to(C.dtype), rtol=1e-2, atol=1e-2),
)

tuner.set_compile_args(
    target='auto',
    execution_backend='auto',
    out_idx=[-1],
)

artifact = tuner.run()             # 编译 + 运行 + 验证所有配置
best_kernel = artifact.kernel      # 最佳 kernel
best_latency = artifact.latency    # 最佳延迟
best_config = artifact.config      # 最佳配置

# 复用最佳 kernel
best_kernel(A, B, C)
```

### 提供输入数据的三种方式（优先级从高到低）

**方式一：上下文管理器（推荐）**
```python
from tilelang.autotuner import set_autotune_inputs

with set_autotune_inputs(A, B, C):
    tuned_kernel = matmul(M, N, K)  # 编译、调优、返回最佳 kernel
```

**方式二：自定义数据提供程序**
```python
def supply_prog(signature):
    return [A, B, C]

tuner.set_profile_args(supply_prog=supply_prog)
```

**方式三：内置生成器（supply_type）**
- `TensorSupplyType.Auto`（默认）：按 dtype 启发式生成
- `Integer`, `Uniform`, `Normal`, `Randn`, `Zero`, `One`

## 十、缓存与持久化

### 缓存机制

TileLang 自动缓存已编译的最佳内核，避免重复编译：

```python
# 相同参数重复调用使用缓存
C1 = matmul(A, B)  # 首次：编译 + 调优
C2 = matmul(A, B)  # 后续：直接使用缓存
```

### 缓存环境变量

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| `TILELANG_CACHE_DIR` | `~/.tilelang/cache` | 编译缓存目录 |
| `TILELANG_TMP_DIR` | `$TILELANG_CACHE_DIR/tmp` | 临时目录 |
| `TILELANG_DISABLE_CACHE` | `0` | 禁用所有内核缓存 |
| `TILELANG_AUTO_TUNING_DISABLE_CACHE` | `0` | 仅禁用自动调优磁盘缓存 |

### CPU 工作线程控制

| 环境变量 | 说明 |
|---------|------|
| `TILELANG_AUTO_TUNING_CPU_UTILITIES` | CPU 使用率（默认 0.9） |
| `TILELANG_AUTO_TUNING_CPU_COUNTS` | CPU 核心数（-1 自动） |
| `TILELANG_AUTO_TUNING_MAX_CPU_COUNT` | 最大 CPU 核心数（-1 不限） |

### 保存与加载调优结果

```python
artifact = tuner.run()  # AutotuneResult

# 保存到磁盘
from pathlib import Path
save_dir = Path('out/best/matmul_1024')
artifact.save_to_disk(save_dir, verbose=True)

# 重新加载
from tilelang.autotuner.param import AutotuneResult, CompileArgs
restored = AutotuneResult.load_from_disk(save_dir, CompileArgs())
best = restored.kernel
best(A, B, C)
```

           C: T.Tensor((M, N), 'float16')):
        ...
    return k

    dict(block_M=64, block_N=128, block_K=32),
    dict(block_M=128, block_N=128, block_K=64),
]
kernels = factory.par_compile(cfgs, num_workers=4)

## 十一、调优最佳实践

1. **从小配置空间开始**：先测试少量配置，确定大致范围，再逐步细化
2. **注意 Shared Memory 限制**：过大的 block_M × block_K 会超出 Shared Memory 容量
3. **线程数对齐**：threads 必须是 32（Warp 大小）的倍数
4. **流水线级数**：num_stages 通常 2~4 为最佳，更多级数消耗更多 Shared Memory
5. **预热充分**：warmup 次数建议 ≥ 10，确保 GPU 达到稳定状态
6. **多次测量**：rep 次数建议 ≥ 50，减少测量噪声
