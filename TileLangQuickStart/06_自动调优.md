# 06 自动调优

TileLang-Ascend 提供了编译优化配置（pass_configs）和自动调优功能，帮助开发者获得最优的内核性能。

---

## 一、编译优化配置（pass_configs）

TileLang-Ascend 通过 `pass_configs` 字典控制编译优化 Pass 的行为。这些配置在 `@tilelang.jit` 装饰器中传入。

### 基本用法

```python
import tilelang

pass_configs = {
    tilelang.PassConfigKey.TL_ASCEND_AUTO_SYNC: True,
    tilelang.PassConfigKey.TL_ASCEND_MEMORY_PLANNING: True,
    tilelang.PassConfigKey.TL_ASCEND_AUTO_CV_COMBINE: True,
}

@tilelang.jit(out_idx=[-1], pass_configs=pass_configs)
def my_kernel(M, N, K, block_M, block_N, K_L1, dtype="float16"):
    ...
```

### 配置项详解

#### TL_ASCEND_AUTO_SYNC — 自动同步插入

```python
tilelang.PassConfigKey.TL_ASCEND_AUTO_SYNC: True
```

**功能**：编译器自动在需要的位置插入 `T.barrier_all()` 等同步指令。

**效果**：
- 开启后，开发者无需手动在每步操作之间插入 `T.barrier_all()`
- 编译器分析数据依赖关系，自动插入必要的同步屏障
- 减少因遗漏同步导致的竞态条件错误

**示例**（开启自动同步前）：

```python
# 需要手动插入 barrier_all
T.copy(A[bx * block_M, k * K_L1], A_L1)
T.copy(B[k * K_L1, by * block_N], B_L1)
T.barrier_all()         # 手动同步
T.gemm_v0(A_L1, B_L1, C_L0, init=(k == 0))
T.barrier_all()         # 手动同步
```

**示例**（开启自动同步后）：

```python
# 编译器自动插入同步，代码更简洁
T.copy(A[bx * block_M, k * K_L1], A_L1)
T.copy(B[k * K_L1, by * block_N], B_L1)
T.gemm_v0(A_L1, B_L1, C_L0, init=(k == 0))
```

---

#### TL_ASCEND_MEMORY_PLANNING — 自动内存规划

```python
tilelang.PassConfigKey.TL_ASCEND_MEMORY_PLANNING: True
```

**功能**：编译器自动进行片上存储的复用规划。

**效果**：
- 分析buffer的生命周期，自动复用不再需要的存储空间
- 减少片上存储的总占用，允许使用更大的block size
- 解决 `T.Parallel` 产生的临时buffer导致的空间浪费

**适用场景**：
- 内核中使用了多个临时buffer
- 使用了 `T.Parallel` 产生的中间结果
- 片上存储空间紧张

---

#### TL_ASCEND_AUTO_CV_COMBINE — 自动Cube/Vector分离

```python
tilelang.PassConfigKey.TL_ASCEND_AUTO_CV_COMBINE: True
```

**功能**：编译器自动将内核中的Cube操作（矩阵乘法）和Vector操作（向量计算）分离到不同的执行核心。

**效果**：
- 自动识别哪些操作属于Cube Core，哪些属于Vector Core
- 自动生成Cube和Vector之间的数据传递和同步代码
- 允许开发者在统一的编程模型中编写融合算子

**适用场景**：
- 融合算子（如GEMM + Bias + ReLU）
- Flash Attention（Cube做QK^T和PV，Vector做softmax）
- 任何需要Cube和Vector协作的场景

**示例**（Developer模式的GEMM + 自动CV分离）：

```python
pass_configs = {
    tilelang.PassConfigKey.TL_ASCEND_AUTO_CV_COMBINE: True,
    tilelang.PassConfigKey.TL_ASCEND_AUTO_SYNC: True,
}

@tilelang.jit(out_idx=[-1], pass_configs=pass_configs)
def matmul(M, N, K, block_M, block_N, K_L1, dtype="float16", accum_dtype="float"):
    m_num = M // block_M
    n_num = N // block_N

    @T.prim_func
    def main(
        A: T.Tensor((M, K), dtype),
        B: T.Tensor((K, N), dtype),
        C: T.Tensor((M, N), dtype),
    ):
        with T.Kernel(m_num * n_num, is_npu=True) as (cid, _):
            bx = cid // n_num
            by = cid % n_num

            # Developer模式：使用alloc_shared/alloc_fragment
            A_L1 = T.alloc_shared((block_M, K_L1), dtype)
            B_L1 = T.alloc_shared((K_L1, block_N), dtype)
            C_L0 = T.alloc_fragment((block_M, block_N), accum_dtype)

            # 无需显式T.Scope("C")，编译器自动识别
            loop_k = T.ceildiv(K, K_L1)
            for k in T.serial(loop_k):
                T.copy(A[bx * block_M, k * K_L1], A_L1)
                T.copy(B[k * K_L1, by * block_N], B_L1)
                T.gemm_v0(A_L1, B_L1, C_L0, init=(k == 0))

            T.copy(C_L0, C[bx * block_M, by * block_N])

    return main
```

## 二、配置组合建议

### 纯Cube操作（GEMM）

```python
pass_configs = {
    tilelang.PassConfigKey.TL_ASCEND_AUTO_SYNC: True,
    tilelang.PassConfigKey.TL_ASCEND_MEMORY_PLANNING: True,
}
```

### 纯Vector操作（Elementwise）

```python
pass_configs = {
    tilelang.PassConfigKey.TL_ASCEND_AUTO_SYNC: True,
    tilelang.PassConfigKey.TL_ASCEND_MEMORY_PLANNING: True,
}
```

### 融合操作（Cube + Vector）

```python
pass_configs = {
    tilelang.PassConfigKey.TL_ASCEND_AUTO_CV_COMBINE: True,
    tilelang.PassConfigKey.TL_ASCEND_AUTO_SYNC: True,
}
```

### 核间流水线

使用核间流水线（Inter-core Pipelined）时，还需额外开启：

```python
pass_configs = {
    tilelang.PassConfigKey.TL_ASCEND_AUTO_CV_COMBINE: True,
    tilelang.PassConfigKey.TL_ASCEND_AUTO_SYNC: True,
    tilelang.PassConfigKey.TL_ASCEND_AUTO_CV_SYNC: True,
}
```

## 三、Expert模式（无自动优化）

在Expert模式下，开发者手动控制所有同步和内存分配，不需要开启自动优化：

```python
# Expert模式：手动同步，手动分配
@tilelang.jit(out_idx=[-1])
def matmul_expert(M, N, K, block_M, block_N, K_L1, dtype="float16", accum_dtype="float"):
    m_num = M // block_M
    n_num = N // block_N

    @T.prim_func
    def main(
        A: T.Tensor((M, K), dtype),
        B: T.Tensor((K, N), dtype),
        C: T.Tensor((M, N), dtype),
    ):
        with T.Kernel(m_num * n_num, is_npu=True) as (cid, _):
            bx = cid // n_num
            by = cid % n_num

            # 显式使用Expert模式内存分配
            A_L1 = T.alloc_L1((block_M, K_L1), dtype)
            B_L1 = T.alloc_L1((K_L1, block_N), dtype)
            C_L0 = T.alloc_L0C((block_M, block_N), accum_dtype)

            with T.Scope("C"):
                loop_k = T.ceildiv(K, K_L1)
                for k in T.serial(loop_k):
                    T.copy(A[bx * block_M, k * K_L1], A_L1)
                    T.copy(B[k * K_L1, by * block_N], B_L1)

                    T.barrier_all()  # 手动同步
                    T.gemm_v0(A_L1, B_L1, C_L0, init=(k == 0))
                    T.barrier_all()  # 手动同步

                T.copy(C_L0, C[bx * block_M, by * block_N])

    return main
```

## 四、性能测试

使用 `tilelang.profiler.do_bench` 进行性能基准测试：

```python
from tilelang.profiler import do_bench

# 编译内核
func = matmul(M, N, K, 128, 256, 64)

# 准备数据
a = torch.randn(M, K).half().npu()
b = torch.randn(K, N).half().npu()

# 基准测试
tilelang_time = do_bench(lambda: func(a, b))
torch_time = do_bench(lambda: a @ b)

print(f"TileLang time: {tilelang_time} ms")
print(f"Torch time: {torch_time} ms")
print(f"加速比: {torch_time / tilelang_time:.2f}x")
```

## 五、block size 调优指南

### GEMM block size 选择

| 参数 | 常用值 | 说明 |
|------|--------|------|
| `block_M` | 128, 256 | 输出Tile的M维度 |
| `block_N` | 128, 256 | 输出Tile的N维度 |
| `K_L1` | 64, 128, 256 | 每次加载的K维度 |
| `block_K` | 32, 64 | L0A/L0B的分形大小 |

### 硬件约束

- UB总使用量 <= 2MB（910B）
- L1总使用量 <= 1MB（910B）
- block_M/block_N/block_K 必须是分形大小（16或32）的整数倍
- 流水线深度 x 单stage Buffer <= Buffer容量

### 选择建议

1. **从默认配置开始**：block_M=128, block_N=256, K_L1=64
2. **逐步调大**：如果片上存储允许，增大block size可以提高数据复用
3. **关注存储容量**：计算总buffer使用量，确保不超出硬件限制
