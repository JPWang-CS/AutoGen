# 05 类型系统

TileLang 提供了丰富的数据类型支持，包括标准浮点、整数类型以及低精度 AI 推理格式。

---

## 一、类型指定方式

在 TileLang 中，dtype 可以通过以下方式指定：

```python
# 方式 1：字符串
T.alloc_shared((128,), "float16")
T.alloc_fragment((32, 32), "float32")

# 方式 2：类型别名
dtype = "float16"
T.alloc_shared((128,), dtype)
```

---

## 二、标量类型

### 2.1 浮点类型

| 类型名 | 位宽 | 说明 | 典型用途 |
|--------|------|------|----------|
| `float64` | 64 位 | 双精度浮点 | 科学计算 |
| `float32` | 32 位 | 单精度浮点 | 通用计算、累加器 |
| `float16` | 16 位 | 半精度浮点 (FP16) | 深度学习训练/推理 |
| `bfloat16` | 16 位 | Brain 浮点 (BF16) | 深度学习训练 |
| `float8_e4m3fn` | 8 位 | FP8 E4M3 | 低精度推理/训练 |
| `float8_e5m2` | 8 位 | FP8 E5M2 | 低精度梯度 |
| `float8_e4m3fnuz` | 8 位 | FP8 E4M3（无偏移） | 特定硬件优化 |
| `float8_e5m2fnuz` | 8 位 | FP8 E5M2（无偏移） | 特定硬件优化 |
| `float6_e3m2fn` | 6 位 | FP6 E3M2 | 超低精度 |
| `float6_e2m3fn` | 6 位 | FP6 E2M3 | 超低精度 |
| `float4_e2m1fn` | 4 位 | FP4 E2M1 | 极低精度 |

### 2.2 整数类型

| 类型名 | 位宽 | 说明 |
|--------|------|------|

| `int16` | 16 位 | 有符号 16 位整数 |
| `int32` | 32 位 | 有符号 32 位整数 |
| `int64` | 64 位 | 有符号 64 位整数 |

## 三、向量类型（Vectorized Types）

TileLang 支持向量化数据类型，用于高效内存访问：

### 语法格式

```
{base_type}x{vector_width}
```

### 支持的向量宽度

| 后缀 | 向量宽度 | 说明 |
|------|---------|------|
| `x2` | 2 元素 | 32 位对齐（对 float16） |
| `x4` | 4 元素 | 64 位对齐 |
| `x8` | 8 元素 | 128 位对齐 |
| `x16` | 16 元素 | 256 位对齐 |

### 示例

```python
# 向量化类型用于高效加载
T.alloc_shared((128,), "float16x8")   # 每个"元素"包含 8 个 float16
T.alloc_shared((64,), "float32x4")    # 每个"元素"包含 4 个 float32
T.alloc_shared((32,), "int8x16")      # 每个"元素"包含 16 个 int8
```

**使用场景**：
- 提高 Global Memory 到 Shared Memory 的拷贝带宽利用率
- 实现 128 位对齐的内存访问
- 通常与 `T.copy` 配合使用，编译器会自动选择最优向量宽度

## 四、Float8 家族详解

### 4.1 float8_e4m3fn

- **格式**：1 位符号 + 4 位指数 + 3 位尾数
- **范围**：约 ±448
- **精度**：3 位尾数
- **用途**：FP8 推理和训练中的前向激活值和权重

### 4.2 float8_e5m2

- **格式**：1 位符号 + 5 位指数 + 2 位尾数
- **范围**：约 ±57344
- **精度**：2 位尾数
- **用途**：FP8 训练中的梯度值

### 4.3 使用 FP8 的 GEMM 示例

```python
@T.prim_func
def fp8_gemm(
    A: T.Tensor((M, K), "float8_e4m3fn"),
    B: T.Tensor((K, N), "float8_e4m3fn"),
    C: T.Tensor((M, N), "float32"),
):
    with T.Kernel(T.ceildiv(N, BN), T.ceildiv(M, BM), threads=128) as (bx, by):
        A_s = T.alloc_shared((BM, BK), "float8_e4m3fn")
        B_s = T.alloc_shared((BK, BN), "float8_e4m3fn")
        C_f = T.alloc_fragment((BM, BN), "float32")

        T.clear(C_f)
        for ko in T.Pipelined(T.ceildiv(K, BK), num_stages=2):
            T.copy(A[by * BM, ko * BK], A_s)
            T.copy(B[ko * BK, bx * BN], B_s)
            T.gemm(A_s, B_s, C_f)

        T.copy(C_f, C[by * BM, bx * BN])
```

## 五、类型转换

### 5.1 T.cast — 显式类型转换

```python
# 单值转换
result = T.cast(value, "float32")

# Fragment 级别转换
for i in T.serial(BN):
    output[i] = T.cast(input[i], "float16")
```

### 5.2 隐式类型转换

在 `T.gemm` 中，TileLang 支持混合精度运算：

```python
# A 和 B 为 float16，C 为 float32 —— 自动处理精度转换
A_s = T.alloc_shared((BM, BK), "float16")
B_s = T.alloc_shared((BK, BN), "float16")
C_f = T.alloc_fragment((BM, BN), "float32")

T.gemm(A_s, B_s, C_f)  # float16 输入 → float32 累加
```

## 六、类型选择建议

| 场景 | 推荐输入类型 | 推荐累加类型 |
|------|------------|------------|
| 训练 | `float16` 或 `bfloat16` | `float32` |
| 推理（精度优先） | `float16` | `float32` |
| 推理（速度优先） | `float8_e4m3fn` | `float32` |
| 量化推理 | `int8` | `int32` |
| 超低精度 | `float4_e2m1fn` | `float32` |
