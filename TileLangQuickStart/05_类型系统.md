# 05 类型系统

本章介绍 TileLang-Ascend 支持的数据类型、类型转换以及精度相关的概念。

---

## 一、基础数据类型

TileLang-Ascend 中使用字符串指定数据类型：

### 支持的类型

| 类型字符串 | 说明 | 字节数 | 典型用途 |
|-----------|------|--------|---------|
| `"float16"` | 半精度浮点 | 2 | 矩阵输入、数据存储 |
| `"float32"` / `"float"` | 单精度浮点 | 4 | 累加器、向量计算 |
| `"bfloat16"` | BFloat16 | 2 | 矩阵输入（低精度训练） |
| `"int8"` | 8位整数 | 1 | 量化推理 |
| `"int16"` | 16位整数 | 2 | 量化计算 |
| `"int32"` | 32位整数 | 4 | 索引、计数器 |
| `"int64"` | 64位整数 | 8 | 大规模索引 |
| `"uint8"` | 8位无符号整数 | 1 | 量化推理 |
| `"uint16"` | 16位无符号整数 | 2 | 量化计算 |
| `"uint32"` | 32位无符号整数 | 4 | 位操作 |
| `"uint64"` | 64位无符号整数 | 8 | 位操作 |

### 类型使用方式

在 TileLang 中，类型通过字符串传递给 `T.Tensor`、`T.alloc_*` 等接口：

```python
# Tensor声明
A: T.Tensor((M, K), "float16")

# 内存分配
a_ub = T.alloc_ub((block_M, block_N), "float32")
C_L0 = T.alloc_L0C((block_M, block_N), "float")
```

### NPU vs GPU 类型指定对比

| 差异点 | NPU (tilelang-ascend) | GPU (上游 TileLang) |
|--------|----------------------|---------------------|
| 类型表示 | 字符串 `"float16"` | 字符串 `"float16"` 或 `T.float16` |
| 半精度 | `"float16"` | `"float16"` 或 `T.float16` |
| BF16 | `"bfloat16"` | `"bfloat16"` 或 `T.bfloat16` |
| 单精度 | `"float"` 或 `"float32"` | `"float"` 或 `"float32"` |
| **推荐** | **仅使用字符串** | 两种均可 |

> **注意**：NPU 上**禁止使用 `T.float16`、`T.bfloat16` 等类型对象**，必须使用字符串形式。这是因为 tilelang-ascend 编译器对类型的解析方式与 GPU 版本不同。

## 二、累加器类型（accum_dtype）

在矩阵乘法中，通常使用混合精度计算：输入为 `float16`，累加为 `float32`。

```python
dtype = "float16"        # 输入数据类型
accum_dtype = "float"    # 累加器数据类型（等价于 "float32"）

A_L1 = T.alloc_L1((block_M, K_L1), dtype)        # float16
B_L1 = T.alloc_L1((K_L1, block_N), dtype)         # float16
C_L0 = T.alloc_L0C((block_M, block_N), accum_dtype)  # float32
```

**为什么使用 float32 累加？**
- 矩阵乘法涉及大量乘加运算，float16 精度容易溢出
- float32 累加保证计算精度，最终结果可转回 float16

## 三、类型转换

### 3.1 T.tile.cast — 精度转换指令

```python
T.tile.cast(dst, src, mode, count)
```

**功能**：根据源操作数和目的操作数的数据类型进行精度转换。

**参数**：

| 参数 | 说明 |
|------|------|
| `dst` | 目标buffer（决定了输出数据类型） |
| `src` | 源buffer（决定了输入数据类型） |
| `mode` | 舍入模式 |
| `count` | 参与计算的元素个数 |

**舍入模式**：

| 模式 | 说明 |
|------|------|
| `"CAST_NONE"` | 无特殊舍入 |
| `"CAST_RINT"` | 四舍五入到最近偶数 |
| `"CAST_FLOOR"` | 向下取整 |
| `"CAST_CEIL"` | 向上取整 |
| `"CAST_ROUND"` | 四舍五入 |
| `"CAST_TRUNC"` | 截断 |
| `"CAST_ODD"` | 向奇数舍入 |

**示例**：

```python
# float32 -> float16
acc_o_half = T.alloc_ub((block_M // 2, dim), "float16")
T.tile.cast(acc_o_half, acc_o, "CAST_RINT", block_M // 2 * dim)
```

### 3.2 通过 T.copy 隐式转换

`T.copy` 在不同类型buffer之间搬运时会自动进行类型转换：

```python
# L0C (float32) -> GM (float16)，自动精度转换
T.copy(C_L0, C[bx * block_M, by * block_N])
```

## 四、T.DataType 工具类

`T.DataType` 提供了数据类型的元信息查询：

```python
from tilelang import DataType

dtype = DataType("float16")
print(dtype.bits)  # 16
print(dtype.bytes)  # 2

# 用于计算地址偏移
offset = block_M * dim * DataType("float16").bits // 8
```

### 常见用法

在 `T.annotate_address` 中用于计算地址偏移：

```python
T.annotate_address({
    q_l1: 0,
    k_l1: block_M * dim * DataType(dtype).bits // 8,
    ...
})
```

## 五、PyTorch数据类型映射

在 Host 端使用 PyTorch 时，需要正确对应数据类型：

| TileLang类型 | PyTorch类型 | NPU创建方式 | GPU创建方式 |
|-------------|------------|------------|------------|
| `"float16"` | `torch.float16` | `torch.randn(M, K).half().npu()` | `torch.randn(M, K).half().cuda()` |
| `"float32"` / `"float"` | `torch.float32` | `torch.randn(M, K).npu()` | `torch.randn(M, K).cuda()` |
| `"bfloat16"` | `torch.bfloat16` | `torch.randn(M, K).bfloat16().npu()` | `torch.randn(M, K).bfloat16().cuda()` |

### 数据准备示例

```python
import torch

# float16 输入 + float16 输出
a = torch.randn(M, K).half().npu()
b = torch.randn(K, N).half().npu()
c = torch.empty(M, N).half().npu()

# float32 输入 + float32 输出
a = torch.randn(M, N).npu()
b = torch.randn(M, N).npu()

# 精度对比
torch.testing.assert_close(c, ref_c, rtol=1e-2, atol=1e-2)
```

### NPU vs GPU 数据放置对比

```python
# NPU: .npu() 放置到昇腾设备
a = torch.randn(M, K).half().npu()

# GPU: .cuda() 放置到 NVIDIA 设备
a = torch.randn(M, K).half().cuda()
```

| 操作 | NPU | GPU |
|------|-----|-----|
| 设备放置 | `.npu()` | `.cuda()` |
| 设备指定 | `torch.npu.device("npu:0")` | `torch.cuda.device("cuda:0")` |
| 设备数量 | `torch.npu.device_count()` | `torch.cuda.device_count()` |
| 张量转移 | `tensor.npu()` / `tensor.cpu()` | `tensor.cuda()` / `tensor.cpu()` |

## 六、精度注意事项

### GEMM精度

GEMM计算使用 float16 输入和 float32 累加，精度验证推荐容差：

```python
torch.testing.assert_close(c, ref_c, rtol=1e-2, atol=1e-2)
```

### Flash Attention精度

Flash Attention 涉及 softmax 计算，中间结果使用 float32 累加：

```python
dtype = "float16"         # Q/K/V输入类型
accum_dtype = "float"     # attention score累加类型
```

### 类型选择建议

| 场景 | 输入类型 | 累加类型 | 说明 |
|------|---------|---------|------|
| GEMM | float16 | float32 | 标准混合精度 |
| Vector Add | float32 | - | 无需累加器 |
| Flash Attention | float16 | float32 | softmax需高精度累加 |
| 量化推理 | int8 | int32 | 量化计算 |
